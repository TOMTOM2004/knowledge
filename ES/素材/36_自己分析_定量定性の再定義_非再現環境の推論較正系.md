# 「定量×定性の掛け合わせ」の再定義 — 非再現環境における推論の較正系

## この文書の位置づけ

明治安田AM・みずほFG の面接で一貫して打ち出してきた「定量分析と定性分析の掛け合わせ」を、面接後に徹底的に言語化し直したもの。#35（定性情報の定量化PoC＝**体験**）から抽出した**思想・理論**側。「定量×定性」を「定性データの構造化」と捉えると本質を取りこぼす、という違和感の正体を解明した記録。

**結論を一行で**：自分がやりたいことは「定性データの構造化」ではなく、**「再現不能（n=1・非定常）な世界で、原因を事実に格上げできないと認めた上で、判断を較正し続ける推論の較正系（calibration engine）」**。

---

## 1. なぜ「定性データの構造化」ではないのか

- **構造化とは、定性を定量に潰す操作**（テキスト→特徴量→数値化）。徹底すると最後は全部「定量」になり、「掛け合わせ」が片側に吸収されて消える。
- 掛け合わせが意味を持つのは、**構造化に抵抗する定性が、構造化されないまま判断に効く**から。価値の核は構造化の手前ではなく、構造化に**抵抗する側**にある。
- 「構造化」と語ると、価値の源泉を自分でコモディティ側（DX・ツール屋）に明け渡してしまう。**飽きられているのは「構造化」と語った版**であって、思想そのものではない。

### 明治安田AM での齟齬の正体
- フレームの問題でなく**共有できる具体的参照先（1社の worked example）の欠如**。「定量×定性」は抽象語で、聞き手が中身を勝手に充填する。
- 修復は抽象的定義の精緻化でなく、**定量と定性が対立した1社の実例**（対立→裁定→結果）。「両方見ました」型は無価値。**対立と裁定があって初めて判断が可視化される**。
- アイデアは本物だが**未成形（operationalize されていない）**だった。それが面接で露出した。

### みずほFG
- 2次：意図が伝わった（定性側・判断としての手応え）。
- 3次：核心は深掘りされず一貫性確認 → **「通過済み論点」化した証拠**であって、価値を置かれなかったのではない。
- ツールとしてでなく**判断として食いついた**。当初の「ツール屋に分類される」懸念は外れていた。

---

## 2. 構造化 ≠ 標準化（混同の解消）

回帰のイメージで分解すると明確：

- **構造化** = 定性を*変数の形*にする（表現の問題・上流）
- **標準化** = 変数を*共通の物差し*に乗せる＝回帰直線を引く（正規化・下流）
- **残差（企業らしさ）** = 標準化の引き算が捨てた、直線からの逸脱

**統合がどこで起きるか**が「構造化する／しない」の本質：
- 構造化する＝定量と定性を混ぜて**一つのモデル**に（統合がデータ層）→ 特徴量追加のトレッドミル＝DXの道
- 構造化しない＝**分けてそれぞれやり、後で結果を統合**（統合が判断層）→ 判断の道
  - ただし「後で統合」が**固定の式**なら実質一モデル。**統合点が判断（可変・定性が定量を上書き可）**でないと分離は見かけだけ。

「定性を構造化する」は、価値の核に対しては**最初から間違った動詞**だった。

---

## 3. 属人性をどう扱うか

- 解釈の核（**その事象がどれだけ影響するか**の判断＝残差因子への重み・符号・確信度の付与）は**原理的に属人**。標準化できたらそれはもう残差でない。
- だが**属人性を消すのでなく、縮めて足場で囲う**：
  - 収集プロトコル → 標準化（聞くこと・深掘りの discipline）
  - どの残差がデカいか（深掘り対象）の検出 → モデルが外れ値フラグで標準化
  - 解釈結果の記録フォーマット → 標準化（監査・伝達・教育可能に）
  - 残るは**解釈の跳躍そのもの**だけ。ゴールは属人ゼロ（＝残差を捨てるDX幻想）でなく、**代替不能な一点に属人性を凝縮し周囲を全システム化**。
- 実務の形：定量スクリーニングが外れ値を出す → アナリストがそこだけ深掘り → 標準メモに書く。**判断は人、プロセスは工業化**。

## 4. LLM をどう使うか — 形式化でなくナレッジ化

混同していた二つの軸（独立）：
- **軸A：統合の場所**（モデル層／判断層）＝構造化する／しない
- **軸B：周辺業務の自動化度**（手作業／LLMで systematic）＝効率・属人性

LLMで業務をシステム化（軸B）しても、最後の統合を別判断に残せば構造化しない（軸A）は成立。**systematic ≠ 融合**。「LLM活用＝結局構造化では」という堂々巡りの正体は、この二軸の混同＋「事例として残す」第三項を忘れていたこと。

### 形式化（parametric）でなくナレッジ化（case-based / RAG）
- 形式化＝重みに畳む＝閉じた条件の表現。ノイズが重みを汚染、文脈が平均化で消える。
- **ナレッジ化＝事例で保持＝開いた条件の表現**。新しい「ただしYの場合」は脆いルール木の書き換えでなく**新事例の追加**で済む。文脈が事例にぶら下がったまま残る。ノイズ事例も「これはノイズ、理由はこれ」として座るだけで汚染しない。
- 暗黙知（感覚）は無構造でなく「通常X、ただしY」の条件構造を持つ。だが**Yが開いている（例外が無限後退）**ため有限ルールに書けない。**開いた条件の正しい表現が case-based**。だから「形式化でなくナレッジ化」が正解。

---

## 5. 仕事の重心は入口（ヒアリング）へ移る

- 分析が LLM でコモディティ化 → 限界価値が「どう分析するか（出口）」から「**誰も持たない入力をどう取るか（入口）**」へ。alpha は proprietary な未デジタル情報の取得に。
- 「**分析で推測どまりの箇所だけを一次情報で確認しに行く**」がヒアリングの定義。確定済みは訊かない。
- オルタナデータ等の自動化は**マイナスでなくスクリーニング**：埋まった分は訊く必要が消え、質問候補から低価値を除外でき、人間のヒアリングを残差の確認に集中させる。
- ヒアリングの durable な核 = (a) 非公開を話させる**関係・信頼**、(b) **何を訊くべきかの選定**。(b) は分析モデルと不可分（標準が分からないと高価値の質問が作れない）。**入口と出口はカップリング**。

---

## 6. 残る唯一の難所 — ノイズからの原因帰属（n=1 問題）

- 市場は **n=1・非再現・非定常**。歴史は一度しか走らず、再実験で signal/noise を分離できない。
- ゆえに「この判断は間違いだった」を**ほぼ永久に事実へ格上げできない**。原因を fact として保存することは市場では嘘になる。**推論に留めるのは謙虚さでなく唯一誠実な表現形式**。
- ナレッジは「fact の DB」でなく「**確信度つき推論の集合**」。
- **resulting（結果で判断品質を測る誤り）への正しい対処**：
  - 規範ラベル（「正しい判断はX」）を**書かない**。残すのは「我々はこう判断し、結果こうなった」＋**確信度つき原因仮説**＋**ex-ante の情報状態と推論**（＝Tetlock の decision journal 作法）。
  - 原因帰属は**事実でなく仮説**として、反証込みで保持。retrieval 時に結果バイアスで偽の原因が伝播するのを防ぐ。
  - ネガティブ結果も事実として残す（file-drawer 回避）。ただし**ex-ante仮説＋方法が精密でないと解釈不能**／**regime 条件を貼る**（timing が悪いだけの健全な approach を永久 blacklist しない）。
- **本当に残る難所はここ一点**。これは自分の思考の穴でなく、**意思決定論全体の未解決フロンティア**。「解ける所は設計で潰し、解けない一点を『解かずに較正し続ける対象』として正しく隔離した」状態。
