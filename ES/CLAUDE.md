# ES フォルダ ルール

エントリーシート（ES）作成・管理の知識ベース。新規ES作成・既存ES編集・エピソード追加の際はこのルールに従う。
ES 作成の全体ワークフロー（es-writer skill・機械チェック・Gate 2/3）は親 `knowledge/CLAUDE.md` を正とする。ここはフォルダ固有ルールのみ。

## フォルダ構成（実構成・2026-06-11 現在）

```
ES/
├── CLAUDE.md            # このファイル
├── changelog.md         # ES提出・変更履歴
├── 部品/                 # 設問カテゴリ別ベストアンサー集
│   ├── best_answers.md  # ★ 頻出設問のベストアンサー（新規ES作成の起点）
│   ├── gakuchika.md     # ガクチカ 全バージョン
│   ├── motivation.md    # 志望動機 企業別
│   ├── self_pr.md       # 自己PR 全バージョン
│   └── other.md         # チームワーク・挫折・弱み・ビジョン等
├── 素材/                 # エピソードライブラリ（STAR形式・35件+）
│   ├── INDEX.md         # ★ エピソード一覧 + 設問別クイックリファレンス（単一ソース）
│   ├── 01〜35_*.md      # 各エピソード
│   └── _draft/          # 整形前草案
├── 企業別/
│   ├── 作成中/           # 下書き（そのまま使わず best_answers と照合してから）
│   └── 提出済/           # 提出した完成版（信頼度高・新規ESのベースとして転用可）
├── エピソード/            # 元の自己分析docx群（参照用・削除変更禁止）
└── 自己分析マインドマップ.md
```

提出後の確定版は `knowledge/移行後ES/` にも集約される（親 CLAUDE.md の配置表参照）。

## 新規ESを書くときの手順（フォルダ固有分）

1. **ベース確認**: `部品/best_answers.md` の該当設問セクション → 詳細は `部品/{gakuchika,motivation,self_pr,other}.md`
2. **エピソード選定**: `素材/INDEX.md` の設問別クイックリファレンスで選ぶ（**設問→エピソードの対応表はINDEX.md が単一ソース**。ここに複製しない）
   - 主力: `03_インターン_財務資料自動化`（ガクチカ最有力）/ `05_塾バイトリーダー`（リーダーシップ系）/ `04_分析設計と失敗`（失敗談・改善力）
3. **企業固有調整**: 「貴社/御社/貴行/貴グループ」等の敬称を合わせる。志望動機は企業固有情報（ビジョン・戦略・シナジー）必須
4. **文字数確認**: `python3 -c "text = '''<本文>'''; print(len(text))"` — 制限の95〜100%を目標

## エピソードを新規追加するとき

`/episode-formatter` skill を使う（草案貼り付け → カテゴリ判定 → STAR形式変換 → `素材/<番号>_<カテゴリ>_<名称>.md` 保存 → INDEX.md 更新まで自動）。

## 管理ルール

- 新規ES提出後は必ず `changelog.md` に提出日・企業名・結果を記録
- `企業別/作成中/` の下書きはそのまま使わず `部品/best_answers.md` と照合して品質を上げてから
- `エピソード/`（docx 原本）は削除・変更禁止（親 CLAUDE.md 禁止事項と同じ）
