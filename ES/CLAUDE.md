# ES フォルダ ルール

このフォルダはエントリーシート（ES）作成・管理のための知識ベース。
新規ES作成・既存ES編集・エピソード追加の際は必ずこのルールに従う。

---

## フォルダ構成

```
ES/
├── CLAUDE.md              # このファイル
├── changelog.md           # ES提出・変更履歴
├── components/            # 設問カテゴリ別ベストアンサー集
│   ├── best_answers.md    # ★ 頻出11設問のベストアンサー（新規ES作成の起点）
│   ├── gakuchika.md       # ガクチカ 全バージョン（Group1〜4）
│   ├── motivation.md      # 志望動機 企業別23件
│   ├── self_pr.md         # 自己PR 全バージョン
│   └── other.md           # チームワーク・挫折・弱み・ビジョン等
├── submitted/             # 提出済みES（信頼度高・ベースとして参照）
├── drafts/                # 下書き中ES
├── md/                    # エピソード・自己分析ライブラリ（構造化済みMD）
│   ├── INDEX.md           # ★ エピソード一覧（設問別クイックリファレンス付き）
│   ├── 01〜15_*.md        # 各エピソード（STAR形式）
│   └── ...
└── エピソード/             # 元の自己分析docxファイル群（参照用）
```

## 新規ESを書くときの手順

### 1. ベースを確認する
まず `components/best_answers.md` を参照し、該当設問のベストアンサーを特定する。

```
設問 → best_answers.md の該当セクション → 詳細は各 components/*.md
```

| 設問 | 参照先 |
|-----|-------|
| ガクチカ | `gakuchika.md`（Group1=インターン ⭐） |
| 志望動機 | `motivation.md` |
| 自己PR・強み | `self_pr.md` |
| チームワーク・挫折・弱み・ビジョン | `other.md` |

### 2. エピソード素材を調べる
`md/INDEX.md` を参照して使えるエピソードを確認する。設問に合うエピソードを選んでから文章化する。

### 3. 企業固有情報を調整する
- 「貴社」「御社」「貴行」「貴グループ」等の敬称を会社に合わせて変更
- 志望動機は企業の固有情報（ビジョン・戦略・シナジー等）を必ず入れる

### 4. 文字数を確認する
```python
python3 -c "text = '''<本文>'''; print(len(text))"
```
制限の95〜100%を目標に調整する（少なすぎる場合はエピソードの詳細を補完）。

## エピソードライブラリの使い方（`md/`）

### 主力エピソード（ガクチカ・強み・失敗談で最優先）
- `03_インターン_財務資料自動化.md` ← **ガクチカ最有力**（16時間→30分、一致率100%）
- `05_バイト_塾バイトリーダー指導品質標準化.md` ← ガクチカ候補（リーダーシップ系）
- `04_インターン_分析設計とコミュニケーション失敗.md` ← 失敗談・改善力

## エピソードを新規追加するとき

`/episode-formatter` スキルを使う。ユーザーが草案を貼り付けると、自動で以下を行う:
1. カテゴリ判定
2. STAR形式に変換
3. `md/<番号>_<カテゴリ>_<名称>.md` として保存
4. `md/INDEX.md` を更新

## 提出済みES・下書きの管理

- **submitted/**: 提出した完成版。品質が高いため、新規ESのベースとして転用可。
- **drafts/**: 作成中・未提出。そのまま使わず `best_answers.md` と照合して品質を上げてから使う。
- 新規ES作成後は必ず `changelog.md` に提出日・企業名・結果を記録する。
