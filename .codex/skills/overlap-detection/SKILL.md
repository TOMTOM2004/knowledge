# overlap-detection

## 目的
複数設問を持つESで、同じエピソード・表現・強みが過度に繰り返されていないかを検出する。

## 使う場面
- ES全設問を見渡して「話がかぶっていないか」確認したいとき
- 新しい設問に回答を追加する前に、既存設問との重複を確認したいとき

## 手順

### Step 1: コンテンツ抽出
各設問回答から以下を抽出する:
- 使用エピソード（いつ・どこで・何をした）
- 言及している強み・能力
- 頻出キーワード（5回以上出現）
- 数字・定量的表現

### Step 2: 重複マトリクス作成

```
| 設問 | EP1 | EP2 | 強みA | 強みB | KW1 |
|------|-----|-----|-------|-------|-----|
| 志望動機 | ✗ | ✓ | ✓ | ✗ | ✓ |
| ガクチカ | ✓ | ✗ | ✓ | ✓ | ✗ |
```

### Step 3: 重複判定
- エピソード重複: 同じエピソードが2設問以上に登場 → 要調整
- 強み重複: 同じ強みが3設問以上で同じ切り口で登場 → 要調整
- キーワード重複: 同一キーワードが1設問内で3回以上 → 要調整

### Step 4: 改善提案
重複が検出された場合:
1. 代替エピソードの候補を `ES/素材/INDEX.md` から探す
2. 同じ強みを別の切り口で表現する方法を提案する
3. 重複キーワードの言い換えを提案する

## 出力形式
- 重複マトリクス（表形式）
- 重複箇所の説明（なぜ問題か）
- 改善案（代替エピソード候補含む）
