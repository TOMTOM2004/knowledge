# data-science-fit

## 前提
digital-common の観点を前提に、データサイエンス職固有の評価基準を追加する。

## データサイエンス職で評価される特性

### 職種固有の評価軸
1. **ビジネス課題からのアプローチ**
   - 「どんなモデルが作れるか」ではなく「何を解決するためにデータを使うか」
   - 技術先行ではなくビジネス起点で考えられるか
2. **実装・本番定着への意志**
   - PoC止まりではなく「実際に動くシステムにする」コミットメント
   - データ品質・運用コスト・精度のバランス感覚
3. **説明可能性への配慮**
   - ブラックボックスにしない（特に金融規制の観点）
   - 非データサイエンティストへの説明能力
4. **継続学習の姿勢**
   - 技術の変化スピードに追いつく自律学習習慣

### ES・面接チェックポイント

**ガクチカ・自己PRで確認すること**
- [ ] データを使って「何かを解決した」経験が示せるか
- [ ] 手法の選定理由を説明できるか（なぜこのモデルを選んだか）
- [ ] 非技術者に説明した経験があるか

**よく聞かれる面接質問**
1. 「今まで携わったデータ分析プロジェクトで最も苦労した点は？」
2. 「機械学習モデルを金融業務に適用する際の課題は？」
3. 「データの前処理で時間を使うことについてどう思いますか？」
4. 「AIの説明可能性（XAI）についてどう考えますか？」
