# interview-qa スキル

## 責務

**このスキルがやること**: 重要度別の回答骨格生成・深掘り展開・詰められ対策・逆質問リスト作成  
**このスキルがやらないこと**: 体験記の収集・質問の重要度分類（→ interview-research が担当）

**前提**: `interview-research` を先に実行し、`interview_research_<ステップ>_*.md` が存在すること

---

## Step 0: 入力確認

以下を確認する。不明な場合はユーザーに質問する。

| 項目 | 例 |
|-----|-----|
| 企業名 | 〇〇証券 |
| コース・職種 | 総合職 / 営業コース |
| 選考ステップ | 1次面接 / 2次面接 / 最終面接 |

---

## Step 1: 入力ファイルの読み込み（並列）

以下を並列で読み込む。

1. `company-info/<企業名>/interview_research_<ステップ>_*.md` — 最新ファイル
   - ファイルが存在しない場合: `【警告】interview_research ファイルが見つかりません。interview-research スキルを先に実行してください。続行しますか？` と表示し、確認を求める
2. `company-info/<企業名>/research_brief.md` — 軸3・軸7-A・軸8・軸9 を抽出
3. `移行後ES/<企業名>.md` — 提出済み回答を確認（存在しない場合はスキップ）
4. `ES/素材/01_自己分析_コア価値観将来像行動原理.md` — コア価値観・行動原理を抽出
5. `ES/素材/02_自己分析_強み弱み伸びしろ.md` — 強み・弱みを抽出

---

## Step 1.5: ES↔面接言い換え対照表（移行後ESが存在する場合のみ）

`移行後ES/<企業名>.md` が存在する場合に自動実行する。存在しない場合はスキップ。

ES内の主要フレーズを抽出し、面接での口語表現に変換する対照表を生成する。

```
### ES↔面接 言い換え対照表

| ES文言 | 面接口語版 | 注意点 |
|--------|----------|--------|
| <ESに書いた表現> | <面接で話す表現> | <ズレやすいポイント> |
```

**抽出基準**:
- 抽象語・カタカナ語・専門用語が含まれるES文言を優先的に抽出する
- 面接口語版は「一読で意味が入る」平易な表現に言い換える（20字以内を目安）
- 注意点には「ESでは〜と書いたが面接では〜と言い換える理由」を1行で記載する

**目的**: ESと面接で言葉が変わることで面接官に「ESと話が違う」と感じさせないための対照表

## Step 2: 重要度別の回答骨格生成

### [高] 質問（フル生成）

重要度「高」の各質問に対して以下をすべて生成する:

```
【高】<質問文>

■ 回答骨格
- 結論: <一言で答え>
- エピソード: <使うエピソードのタイトル>（ES/素材/INDEX.md の番号）
- 具体内容: <数字・固有名詞・アクション>
- 締め: <この企業・コースに接続する一言>

■ 時間別バリエーション
【30秒版】（約75字）
「<結論1文>。<数字を含む根拠1つ>。だから<企業・コースへの接続>。」

【60秒版】（約150字）
「<結論1文>。<エピソードを2〜3文で要約>。この経験から<企業・コースへの接続>。」

> フル版は箇条書きのみ。30秒・60秒版は文章形式で出力する（暗唱練習のため）

■ 自己接続チェック
- この企業の何と繋がるか: <research_brief.md 軸7-A または軸3 の記述>
- エピソードがES提出済みの場合: <面接での言い換えポイント>

■ 深掘り展開（2段）
1段目: <想定深掘り質問> → <回答の方向性>
2段目: <さらなる深掘り質問> → <回答の方向性>

■ 反論耐性チェック
- 想定される反論: <面接官が詰めてくるポイント>
- 対策: <切り返し方の方向性>

■ 地雷回避メモ（interview_research の「地雷回答集」参照）
- ⚠ <この質問に関連する地雷パターン>
- ⚠ <面接官タイプ別の注意点>
> 地雷回答集にデータがない場合は `【地雷データなし】` と表示

■ 企業固有キーワード具体例固定メモ（⑤）
（research_brief.md の軸3・軸7-A に企業理念・固有キーワードが含まれる場合のみ生成）

- 企業が使う固有キーワード（例：「お客様ファースト」「誠実」「地域密着」等）に言及する場合、
  具体例を **1〜2個に絞って固定** する
- 形式: 「〇〇という点で△△を実感しました」（体験談または観察事実に紐づける）
- 抽象的な同意（「〇〇という理念に共感しました」のみ）は禁止。必ず具体事実を1つ添える
- `【キーワード具体例: <キーワード> → <具体事実>】` タグで明示し、面接全体を通じて同じ例を使い回す
```

### [中] 質問（標準生成）

重要度「中」の各質問に対して以下を生成する:

```
【中】<質問文>

■ 回答骨格
- 結論: <一言で答え>
- エピソード: <使うエピソードのタイトル>
- 具体内容: <数字・固有名詞・アクション>
- 締め: <この企業・コースに接続する一言>

■ 深掘り展開（2段）
1段目: <想定深掘り質問> → <回答の方向性>
2段目: <さらなる深掘り質問> → <回答の方向性>
```

### [低] 質問（簡略生成）

重要度「低」の各質問に対して以下のみ生成する（深掘りは省略）:

```
【低】<質問文>
- 結論: <一言で答え>
- 使うエピソード: <タイトル>
- 要点: <2〜3行>
```

### 志望動機系回答の特別ルール

志望動機・「なぜこの企業か」・「なぜこのコースか」に関する質問には、
`research_brief.md` の軸7-A（核心固有性）を必ず組み込む。

```
志望動機の構成:
1. 核心固有性（軸7-A）を起点にする
2. 自分のエピソード・価値観との接続
3. このコースでやりたいこと（軸2・軸5を参照）
```

軸7-A が research_brief.md に存在しない場合:
`【固有性未抽出】company-researcher スキルの軸7-A を実行してから再実行することを推奨します。`
と表示し、続行確認をする。

---

### 複数コース・職種出願時の追加ルール

**適用条件**: research_brief.md または入力情報から、出願コース・職種が2つ以上確認される場合に必ず実行する。

#### 第一志望コース確定版回答の生成（①）

「第一志望はどのコースですか」「どのコースを第一志望にしていますか」系の質問に対して、
確定版回答を必ず生成する（重要度に関わらず必須）。

```
■ 第一志望コース確定版回答
- 第一志望コース: <コース名>
- 理由（30秒版）: 「<コース名>です。理由は<自分のやりたいこと・強みとの合致>だからです。」
- 根拠（補足用）: <他コースとの優先順位の根拠を1〜2文>
```

#### 複数コース併願の一貫ロジック（②）

「なぜ複数コースを受けているのか」の説明ロジックを生成する。

```
■ 複数コース併願ロジック
- 共通軸: <各コースに共通する自分の関心・価値観（1文）>
- 各コースのアプローチの違い:
  - <コースA>: <このコースで実現できること>
  - <コースB>: <このコースで実現できること>
- 優先順位の根拠: <どのコースが自分の強みを最も活かせるか（1〜2文）>
```

**矛盾チェック**: コースAとコースBで「やりたいこと」が相反していないかを検証し、
矛盾がある場合は `【要調整: コース間の矛盾 — <矛盾内容>】` を付与する。

---

### 自己評価型質問の専用フレーム（④）

**適用条件**: interview_research で `【自己評価型】` タグが付いた質問に自動適用する。

```
■ 自己評価型質問の回答骨格

1. 評価（数値または段階を明示）: <GPAなら実数値 / 「平均より上/下」等でも可>
2. 満足している点: <具体的な取り組み・成果を1〜2文>
3. 満足していない点・改善余地: <正直な自己評価を1文>
4. この評価が示す自分の特性: <企業の求める人材像との接続を1文>
```

**禁止パターン**:
- 数値を聞かれているのに「充実していました」と定性で逃げる
- 満足度が低い場合に言い訳から始める
- 満足度が高い場合に謙遜しすぎて具体性を失う

## Step 2.5: 志望動機接続マップ

### 生成条件
志望動機系の質問（「なぜこの業界」「なぜこの会社」「なぜこのコース」「入社後にやりたいこと」）が
重要度「高」または「中」に1つ以上存在する場合に必ず生成する。

### 出力フォーマット

#### 4軸接続表
| 軸 | 回答の核（1〜2文） | 前軸からの接続 | 接続評価 |
|---|---------------|------------|--------|
| なぜこの業界 | | — | |
| なぜこの会社 | | ↑ から自然に繋がるか | |
| なぜこのコース | | ↑ から自然に繋がるか | |
| 入社後にやりたいこと | | ↑ から自然に繋がるか | |

接続評価の凡例: `OK` / `要調整`（具体的な修正案を記載） / `未設定`（対応質問が低または存在しない）

#### 矛盾・重複・飛躍チェック
| チェック観点 | 状態 | 対策 |
|-----------|-----|-----|
| 業界理由と会社理由が同じ内容になっていないか | | |
| 競合他社でも通用する志望動機になっていないか（軸7-A 参照） | | |
| やりたいことが業務実態と乖離していないか（research_brief.md 軸2参照） | | |

## Step 3: 詰められポイントと対策

以下10観点で「詰められやすいポイント」と「切り返し方針」を生成する。

| # | 詰められポイント | 切り返し方針 |
|---|-------------|------------|
| 1 | 志望動機の本気度 | |
| 2 | なぜ競合他社ではないか | |
| 3 | ガクチカの深掘り（数字・役割・貢献） | |
| 4 | 弱みの実在性 | |
| 5 | キャリアプランの具体性 | |
| 6 | このコースでなければいけない理由 | |
| 7 | 困難をどう乗り越えたか | |
| 8 | チームでの役割・貢献の再現性 | |
| 9 | 学業・専攻との繋がり | |
| 10 | 入社後のイメージの具体性 | |
| 11 | 志望度・他社選考状況・内定時の意思決定 | 切り返し方針: ① 「第一志望か」→ 明確にYes/Noを言い切る（曖昧な逃げは地雷）② 「他社の状況は？」→ 受けている企業群の共通軸を説明し、この企業が最上位である根拠を添える ③ 「内定が出たらどうするか」→ 意思決定基準（何を軸に選ぶか）を事前に言語化。構成：「判断軸→この企業が基準を満たす理由」 ④ 「なぜまだ内定がないのか」→ 選考の時期・段階を客観的に説明し、ネガティブに受け取らせない |

## 品質ガードレール

- 重要度「高」・「中」の質問には深掘り2段展開を必須とする
- 重要度「低」の質問は深掘り省略（骨格のみ）
- 志望動機系回答には軸7-A の核心固有性を必ず組み込む
- 軸7-A が未抽出の場合は `【固有性未抽出】` タグをつけ company-researcher 再実行を推奨する
- 回答骨格は箇条書きのみ（文章化しない）—— 暗記を防ぐため
- interview_research ファイルなしで実行する場合は警告を出してから続行確認する
- 重要度「高」の質問には「地雷回避メモ」を必須とする（地雷データがない場合も `【地雷データなし】` を出力する）
- 志望動機系の質問が重要度「高」または「中」に存在する場合は「志望動機接続マップ」の生成を必須とする
