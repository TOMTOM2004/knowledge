# strengths-weaknesses-patterns

## 目的
強み・弱み・自己PR設問で「言いっぱなし」にならず、根拠・具体例・職種との対応関係まで示せる構造を作る。

## 強み設問の構造

```
[強みの定義（1文）]
→ [その強みが発揮された具体的な場面・エピソード（2〜3文）]
→ [その結果（数字・評価）]
→ [職種・業務でどう活かすか（1〜2文）]
```

### 強みの「定義」の書き方
- 「コミュニケーション力がある」→ 曖昧すぎる
- 「立場の異なる相手との合意形成ができる」→ 具体的で職種との対応関係がある

**強みのネーミング改善例**:
| 曖昧な表現 | 具体的な表現 |
|-----------|-----------|
| リーダーシップ | 目標から逆算してチームを動かす力 |
| 分析力 | 数値データと定性情報を統合して判断する力 |
| コミュニケーション力 | 相手のニーズを引き出しながら合意を形成する力 |
| 粘り強さ | 成果が出るまで仮説と検証を繰り返す継続力 |

## 弱み設問の構造

```
[弱みの定義（1文）- 正直に、でも致命的でない弱みを選ぶ]
→ [弱みが顕在化した具体的な場面（1〜2文）]
→ [その弱みへの対処・改善策（2文）]
→ [現在の状況（改善中・改善済み）]
```

### 弱みの選び方
- **避けるべき弱み**: 志望職種の本質的能力に直結するもの
  - 例: 法人営業志望なのに「人と話すのが苦手」
- **選ぶべき弱み**: 改善努力が示せる、致命的でない弱み
  - 例: 「完璧主義で締め切り直前まで提出をためらいがち」
  - 例: 「最初から全体像を把握したいため、スタートが慎重になる」

### NGパターン
- 「弱みはありません」（自己認識の欠如）
- 「頑張りすぎること」（強みをそのまま言い換えただけ）
- 弱みだけ言って対処法を述べない

## 自己PR設問との違い
- 強み設問: 1つの強みを深掘り
- 自己PR設問: 自分全体を売り込む（強み+エピソード+志望理由の入口）
