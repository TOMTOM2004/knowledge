# interview-probe-generation

## 目的
ESの主張・エピソード・論理の弱点を特定し、面接での深掘り質問に変換する。

## 使う場面
- ES完成後に「面接で詰められそうな箇所」を把握したいとき
- skeptical-interviewerエージェントの内部手順として使う

## 深掘り質問の生成ルール

### Rule 1: 「言いっぱなし主張」の質問化
ESに根拠なく書かれている主張をすべて疑問文にする。

例:
- 「チームをまとめた」→「チームがバラバラになっていた理由は何ですか？なぜあなたがまとめる立場になったのですか？」
- 「分析力がある」→「その分析力は他者と比べてどの点が優れていると思いますか？証拠は？」

### Rule 2: 数字・具体性の確認質問
数字が登場している箇所には、計算根拠・達成方法を聞く。

例:
- 「売上20%向上」→「20%という数字はどうやって出したのですか？」
- 「50名の組織」→「50名全員と関わったのですか？実際に影響を与えた人数は？」

### Rule 3: 志望動機の本気度テスト
- 「競合他社でも同じことができませんか？」
- 「当社が第一志望でない可能性はありますか？」
- 「入社後、配属が希望通りでなかった場合は？」

### Rule 4: 矛盾・論理の穴の発見
ESを読んで以下を確認:
- 強みとエピソードが対応しているか
- 志望動機とキャリアビジョンが整合しているか
- ガクチカと自己PRで異なる人物像になっていないか

矛盾が見つかった場合→「あなたは〇〇とも書いていましたが、△△との整合性はどう考えますか？」

## 質問難易度の設定

| レベル | 説明 | 使用場面 |
|------|------|---------|
| L1: 確認 | 事実確認・詳細確認 | 1次面接 |
| L2: 深掘り | 根拠・理由の追求 | 2次面接 |
| L3: 反論 | 主張への反証・代替案の提示 | 最終面接 |

## 出力形式
- 質問リスト（カテゴリ別・難易度付き）
- 各質問の「面接官の意図」
- 回答するときの注意点
