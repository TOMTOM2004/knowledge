# interview-research スキル

## 責務

**このスキルがやること**: 選考調査・質問収集・重要度分類・面接実態の把握  
**このスキルがやらないこと**: 想定問答の生成・回答骨格の作成（→ interview-qa が担当）

---

## Step 0: 入力確認

以下を確認する。不明な場合はユーザーに質問する。

| 項目 | 例 |
|-----|-----|
| 企業名 | 〇〇証券 |
| コース・職種 | 総合職 / 営業コース |
| 選考ステップ | 1次面接 / 2次面接 / 最終面接 |

---

## Step 1: 社内ナレッジ確認

以下を並列で読み込む。

1. `company-info/<企業名>/research_brief.md` — 軸3（求める人材像）・軸7-A（核心固有性）・軸9（面接対策）を抽出
2. `company-info/<企業名>/interview_research_<ステップ>_*.md` — 既存ファイルがあれば確認し、上書きするか新規作成するかを確認する
3. `company-info/<企業名>/interview_prep_*.md` — 旧フォーマットがあれば参考情報として参照（上書き禁止）

---

## Step 2: Web調査（4グループ並列実行）

### グループA: 体験記から実際に聞かれた質問を収集

**目的**: 「実際に聞かれた質問」を逐語的に収集する（意訳・要約禁止）

**検索クエリ**:
```
<企業名> <コース> <N>次面接 体験記 質問
<企業名> <コース> <N>次面接 ONE CAREER
<企業名> <コース> 面接 就活会議
<企業名> <コース> 面接 マイナビ 体験談
<企業名> <コース> 面接 みん就
```

**収集基準**:
- 体験記内の「聞かれた質問」をそのまま引用する
- 複数件の体験記で同じ質問が出ていれば「複数件確認」と記録する
- 出典URL・投稿時期を記録する

---

### グループB: 面接形式の調査

**検索クエリ**:
```
<企業名> <N>次面接 形式 時間 人数 オンライン 対面
<企業名> 面接 グループ 個人 ケース
```

**収集内容**:
- 面接形式（個人/グループ/ケース面接等）

- 面接官の人数
- 対面/オンライン
- 質問の件数・スタイル（雑談型/圧迫型/深掘り型等）

---

### グループC: 面接官の特徴調査

**検索クエリ**:
```
<企業名> 面接官 年次 部署 雰囲気
<企業名> <N>次面接 面接官 どんな人
```

**収集内容**:
- 面接官の年次・役職（若手/中堅/役員等）
- 所属部署（人事/現場/経営企画等）
- 面接スタイル（深掘り重視/雰囲気重視/形式通り等）

---

### グループD: 評価軸・求める人材像の実態調査

**検索クエリ**:
```
<企業名> <N>次面接 評価軸 合格 ポイント
<企業名> <コース> 求める人材 実態 合格者
<企業名> ES 通過 特徴 傾向
```

**収集内容**:
- 合格者・内定者の特徴（体験記・口コミ）
- 不合格になった理由（体験記）
- 公式の「求める人材像」との乖離

---

## Step 3: 質問の重要度分類

収集した質問を以下の3段階で分類する。

### 重要度: 高（体験記複数件で確認済み）

**定義**: 体験記の **2件以上** で同じ質問が確認できた質問

**出力形式**:
```
【高】<質問文（逐語引用）>
出典: <URL1>（<年月>）/ <URL2>（<年月>）
```

### 重要度: 中（体験記1件またはパターン予測）

**定義**: 体験記の1件で確認できた質問、または体験記のパターンから予測できる質問

**出力形式**:
```
【中】<質問文>
根拠: <体験記1件の場合: 出典URL> / <パターン予測の場合: 「〇〇という質問が複数あるため関連が予測される」>
```

### 重要度: 低（AI予測・体験記根拠なし）

**定義**: 体験記に根拠がなく、AIが追加予測した質問

**出力形式**:
```
【低】<質問文>
根拠: AI予測（体験記根拠なし）
```

**重要ルール**:
- 体験記が1件しかない質問は「高」ではなく「中」にする
- 体験記が見つからなかった場合は `【体験記なし】` タグを明示し、全質問を「低」にする
- 「実際に聞かれた質問」は意訳・要約しない（逐語引用）

## Step 6: 保存

承認後、以下のパスに保存する:

```
company-info/<企業名>/interview_research_<ステップ>_<YYYYMMDD>.md
```

例: `company-info/〇〇証券/interview_research_1次面接_20260403.md`

保存後:
```
保存完了: company-info/<企業名>/interview_research_<ステップ>_<YYYYMMDD>.md

次のステップ:
「<企業名>の<N>次面接の想定問答を作って」→ interview-qa スキルを使う
```

---

## 品質ガードレール

- 重要度「高」は複数ソースで確認したものだけ（1件のみなら「中」）
- 体験記が見つからない場合は `【体験記なし】` を明示し、全質問を「低」として出力する
- 「実際に聞かれた質問」は意訳・要約しない（逐語引用）
- 体験記の古さが2年以上の場合は `【情報古い可能性あり】` タグをつける
- interview_prep_*.md（旧フォーマット）は参照のみ（上書き・削除禁止）
