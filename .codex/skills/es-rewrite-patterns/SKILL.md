# es-rewrite-patterns

## 目的
ESの表現改善における典型的なリライトパターンを提供する。
editor-refinerエージェントが参照する改善テンプレート集。

## パターン一覧

### Pattern 1: 結論先出し型（PREP法）

**Before（問題）**:
「私は大学のゼミで研究を行い、チームで協力しながら様々な困難を乗り越えてきました。その経験を通じて、主体的に行動することの大切さを学びました。貴社でもこの強みを活かしたいと思います。」

**After（改善）**:
「私の強みは、目標に向けてチームを巻き込んで行動できることです。大学のゼミで○○の研究に取り組んだ際、△△という課題に直面しました。私は□□というアプローチで解決し、結果として○○という成果を出しました。」

**適用場面**: 冒頭3行以内に結論がない文章

---

### Pattern 2: 抽象→具体化型

**Before（問題）**:
「リーダーとして積極的にチームをまとめ、様々な問題に対処しながら成果を上げました。」

**After（改善）**:
「15名のチームリーダーとして、週次MTGを主導し、進捗遅延の早期発見と対処を行いました。3ヶ月で目標の120%を達成しました。」

**変換ルール**:
- 「様々な」→ 具体的な数や種類に置き換える
- 「積極的に」→ 具体的な行動（毎週○○をした等）に置き換える
- 「大きな成果」→ 数字・比率・順位に置き換える

---

### Pattern 3: 数字挿入型

**Before（問題）**:
「多くのお客様と対話し、高い評価をいただきました。」

**After（改善）**:
「月間100件以上のお客様と対話し、顧客満足度調査で部内トップ評価をいただきました。」

**数字化できる要素**:
- 人数（チームメンバー・顧客数・参加者数）
- 期間（週次・月次・○ヶ月で）
- 比率・達成率（目標の○%達成）
- 順位（○名中△位）
- 回数（○回・○件・○社）

---

### Pattern 4: 長文分割型

**Before（問題）**:
一文が70字を超えており、「〜で、〜した結果、〜となり、最終的には〜という経験を通じて〜を学びました。」のような構造。

**分割ルール**:
- 接続詞（そして・また・さらに・その結果）の直前で切る
- 1文は50字以内を目標にする
- 因果関係を「なぜなら」「その結果」で明示する

---

### Pattern 5: 技術語言い換え型

**変換対応表**:
| 専門語 | 文系向け言い換え |
|-------|--------------|
| KPIの達成 | 設定した数値目標の達成 |
| アジャイル | 短サイクルで改善を繰り返す開発 |
| AUM拡大 | 運用資産の増加 |
| バリュエーション | 企業価値の評価・算定 |
| ソリューション提案 | 課題に合わせた解決策の提案 |
| スキームの構築 | 仕組みの設計・構築 |

---

### Pattern 6: 字数調整型

#### 200字 → 300字の増量
- 数字・具体例を1つ追加
- 「なぜそう思ったか」の理由を1文追加
- 成果の背景（チーム・環境）を補足

#### 400字 → 300字の削減
- 背景・前置きを半分に削る
- 「〜という経験から〜を学びました」などの定型締めを削除
- 接続詞・副詞の削除（「また」「さらに」「特に」）
