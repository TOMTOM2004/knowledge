# gakuchika-answer-patterns

## 目的
ESに書いたガクチカを「面接での話し言葉」に変換し、深掘りに耐える構造を設計する。

## 時間別バージョン設計

### 1分バージョン（冒頭の自己紹介・短く聞かれた時）
```
[結論：何をした人か]（10秒）
[課題と行動]（30秒）
[結果と学び]（20秒）
```

### 3分バージョン（標準的なガクチカ質問）
```
[結論：何をした人か + 規模感]（20秒）
[状況・課題の背景]（30秒）
[自分の具体的な行動 ×2〜3点]（60秒）★ここが核心
[結果（数字あれば）]（20秒）
[学び・今後への接続]（30秒）
```

### 5分バージョン（最終面接・深掘りが予想される場面）
3分版を土台に、以下を追加:
- なぜその方法を選んだか（判断の根拠）
- うまくいかなかったこととその対処
- 他のメンバーとの関係・巻き込み方

## 深掘りへの備え（「なぜ×5段」の準備）

例: 「リーダーとしてチームをまとめた」という主張の場合
1. なぜリーダーを引き受けたのか
2. なぜその課題に着目したのか
3. なぜその解決策を選んだのか
4. なぜメンバーが動いてくれたと思うか
5. なぜその経験が成長につながったと思うか

## ガクチカの弱点と話し言葉での補強

| 弱点 | 補強の仕方 |
|-----|---------|
| 数字がない | 「具体的な数字は持っていないのですが、○○という変化がありました」 |
| 役割が不明確 | 「チームの中での私の役割は〜でした」と明示 |
| 活動の説明に終始 | 「特に私が注力したのは〜という点です」で行動を際立たせる |
