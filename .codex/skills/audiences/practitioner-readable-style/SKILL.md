# practitioner-readable-style

## 対象読者プロファイル
- 志望職種の業務を10年以上経験している現場のプロ
- 「業務を分かっているか」「うちのチームで使えるか」を評価する
- 表面的な言葉より「本質を理解しているか」を見抜く
- 2次〜最終面接で登場することが多い

## 評価軸と対処

### 現場が「わかっていないな」と感じるパターン
| 表面的な表現 | 現場が求める表現 |
|------------|--------------|
| 「運用に携わりたい」 | 「アクティブ運用のファンドマネージャーとして○○セクターを担当したい」 |
| 「DXを推進したい」 | 「業務要件定義から実装まで自社で担う内製開発チームに携わりたい」 |
| 「法人営業で活躍したい」 | 「中堅企業の事業承継・M&Aニーズを起点に財務ソリューションを提案したい」 |

### 現場が評価するポイント
1. **業務理解の具体度**: 日次業務・アウトプット・KPIを理解しているか
2. **論理的思考の証拠**: 「なぜそう考えたか」の構造が見えるか
3. **課題設定力**: 言われた通りではなく自分で課題を設定した経験があるか
4. **専門用語の正確な使用**: 専門語を使うなら正確に使う（間違えると逆効果）

### 専門用語の使い方原則
- 正確に理解している用語のみ使う
- 使う場合は定義・文脈を正確に
- 曖昧な場合は使わずに言い換える
- 「使えることを見せようとした感」は逆効果

### 文章スタイル調整
- 結論→根拠→事実の論理構造で書く
- 「〜と思います」より「〜と考えます」「〜と判断しています」
- 自分の判断・選択の根拠を必ず述べる
- 曖昧な一般論を排除し、固有の経験・知識を示す

### 確認チェックリスト
- [ ] 業務内容を具体的に理解した上で書いているか
- [ ] 専門用語を正確に使っているか（間違えていないか）
- [ ] 自分の判断・行動の根拠が論理的に示されているか
- [ ] 「教えてもらえれば何でもできます」ではなく「この点は既に持っている」が示せているか
