# question-bank-updater スキル

## 責務

**このスキルがやること**: transcript から質問を抽出・分類・重複排除・question_bank.md へのマージ  
**このスキルがやらないこと**: 回答骨格の生成（→ interview-qa が担当）・パターン集計（→ interview-reflect が担当）

---

## Step 0: 入力確認

対象 transcript ファイルパスを確認する。

ファイルパスが指定されない場合は、全社の transcript を検索する:
```
company-info/*/transcript_*.md
```
処理済みログ（`docs/question_bank.md` の「処理済み transcript ログ」）に記録されているファイルは除外する。

---

## Step 1: transcript の読み込みと質問抽出

対象 transcript ファイルを読み込み、面接官の発言から質問のみを抽出する。

**質問の抽出基準**:
- 「？」で終わる文
- 面接官（インタビュアー側）の発言として記録されている行
- 「教えてください」「聞かせてください」で終わる依頼形

**抽出しないもの**:
- 受験者の発言
- 面接官の相槌・コメント（「なるほど」「そうですか」等）
- 手続き的な発言（「では始めましょう」「時間になりました」等）

---

## Step 2: 質問の分類

抽出した質問を以下のカテゴリに分類する。

| カテゴリ | 分類キーワード |
|---------|------------|
| 自己紹介・自己PR | 「自己紹介」「強み」「長所」「一言で」 |
| 志望動機 | 「なぜ」「志望」「入社したい理由」「この会社」 |
| ガクチカ・エピソード | 「力を入れた」「経験」「エピソード」「取り組んだ」 |
| キャリア・ビジョン | 「将来」「キャリア」「やりたいこと」「10年後」 |
| 深掘り・詳細確認 | 「具体的には」「それはなぜ」「もう少し」 |
| 業界・企業理解 | 「業界」「会社」「ビジネス」「競合」 |
| 弱み・課題 | 「弱み」「短所」「改善」「失敗」 |
| 逆質問 | 「何か質問は」「聞いておきたいことは」 |

---

## Step 3: 既存 question_bank.md との重複確認

`docs/question_bank.md` を読み込み、抽出した質問と照合する。

**重複判定の基準**:
- 完全一致: 同じ質問文 → 「登録済み」としてスキップし、出現回数を更新
- 意味の類似: 同じ内容で表現が異なる → 「類似質問」として既存質問に別表現を追記し出現回数を更新
- 新規: 既存にない質問 → 新規登録

---

## Step 4: プレビュー提示

question_bank.md への追加内容をプレビューする。ファイルにはまだ保存しない。

```
## question_bank.md 更新プレビュー

処理 transcript: <ファイルパス>
抽出質問数: N件
→ 新規登録: X件 / 類似登録（出現回数更新）: Y件 / 重複スキップ: Z件

### 新規登録する質問

| カテゴリ | 質問文 | 出典（企業・ステップ） |
|---------|------|-----------------|
| ガクチカ | <質問> | <企業名>_<N>次 |

### 類似登録する質問（既存質問の出現回数を更新）

| 既存質問 | 新規類似質問 | 出典 |
|---------|----------|-----|
| <既存質問> | <類似質問> | ... |
```

ユーザーに確認を求める。承認後 Step 5 へ進む。

---

## Step 5: 保存

承認後、`docs/question_bank.md` を更新する。

- 新規質問を該当カテゴリのテーブルに追加する
- 類似質問は既存行に「別表現: <類似質問文>（出典）」として追記し出現回数を更新する
- 処理済み transcript ログに今回のファイルパス・処理日・新規登録数を追記する

```
保存完了: docs/question_bank.md

新規登録: X件 / 類似登録: Y件
次のステップ: 全社パターン更新 → interview-reflect を実行
```

---

## 品質ガードレール

- 質問文は逐語で記録する（要約・意訳禁止）
- 受験者の発言は question_bank に含めない
- 重複判定は「意味の類似」まで考慮する（表記ゆれのみで新規登録しない）
- `docs/question_bank.md` の既存内容は削除しない（追記・更新のみ）
