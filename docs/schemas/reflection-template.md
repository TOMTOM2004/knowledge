# reflection ファイル テンプレート

面接終了後に作成する振り返りファイルのフォーマット。
Gemini等で壁打ちしながら当日中に作成・push する。

---

## ファイル配置

```
company-info/<企業名>/reflection_<N>次_<YYYYMMDD>.md
```

例: `company-info/〇〇証券/reflection_1次_20260403.md`

---

## frontmatter 記入ガイド

| フィールド | 値 | 記入タイミング |
|-----------|---|-------------|
| `result` | `pending` → 合否判明後に `passed` または `rejected` に更新 | 作成時は `pending` |
| `stage` | `1次` / `2次` / `最終` / `内定` | 作成時に記入 |
| `date_result` | 合否連絡が来た日（YYYY-MM-DD） | 合否判明後に記入 |
| `notes` | 一行メモ（例: 「志望動機が薄いと指摘された」） | 任意 |

---

## 作成のコツ

1. 面接終了後30分以内に書く（記憶が新鮮なうちに）
2. 「聞かれた質問」は思い出せる限り逐語で書く（transcript との照合のため）
3. 合否が判明したら必ず frontmatter の `result` と `date_result` を更新する
4. transcript が届いたら `interview-blindspot` を実行して盲点セクションを追記する
