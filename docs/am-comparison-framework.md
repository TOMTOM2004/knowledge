# アセットマネジメント会社 競合比較フレームワーク

## 比較の原則

> **"取り組みの有無"ではなく"価値創出の仕組みの違い"を見る**

エンゲージメント、インベストメントチェーン、顧客本位、責任投資のような項目は、
今のAM会社では多くが掲げる。差が出るのは:

- それを**どう運用判断に接続しているか**
- どこで**利益の源泉（edge）に変えているか**
- それを**組織として再現できるか**

「立派に見える言葉」の比較をしてはいけない。
比較すべきは、**その言葉が実際にどの変数に落ちているか**。

---

## 3層モデル

| 層 | 内容 | 見るもの |
|---|------|---------|
| **1. 表の説明層** | 会社が何を言っているか | 運用哲学、エンゲージメント方針、顧客本位、サステナビリティ方針 |
| **2. 実際の運用プロセス層** | 何を根拠に銘柄を見て、どう意思決定するか | **本当の差別化ポイント** |
| **3. 組織・事業モデル層** | その運用を継続的に回せるのか | 顧客基盤、販売チャネル、親会社関係、収益構造、人材構成 |

---

## 比較の中心的な問い

> **この会社は、どの顧客に、どの領域で、どの収益構造で、今後も勝ち続けられるのか**

追加項目もこの問いに直接効くものを優先する。

## 比較軸一覧（4群・28軸）

### A群: 事業・顧客・商品

| # | 軸 | 優先度 | 見るべき観点 |
|---|---|:---:|------------|
| A-1 | **顧客構成（Client mix）** | 最優先 | 年金・機関投資家中心か / リテール中心か / グループ内販売依存か / 外部販社比率 / 顧客が求めるのはalpha（超過収益）か安定性か商品供給力か |
| A-2 | **販売チャネル構造** | 最優先 | 生保系AMは親会社の販売網に依存するか / 独自で証券会社に卸すか / BtoBtoC構造のどこに強みがあるか |
| A-3 | **グループ内役割分担** | 最優先 | AM子会社 vs 本社運用部門 vs IT子会社の棲み分け / 本社一般勘定の受託比率 |
| A-4 | **母体の法的形態と受託者構造** | 最優先 | 相互会社（契約者に責任）vs 株式会社（株主に責任）/ 「誰のために運用するか」の原理の違い |
| A-5 | **パッシブ/アクティブ比率** | 最優先 | 数字で比較。「アクティブ中心」の定性表現では不十分 |
| A-6 | **旗艦商品（Flagship product）** | 最優先 | 会社が本当に力を入れているのは理念ではなく収益の柱に出る |
| A-7 | **Alternative positioning（オルタナティブの位置づけ）** | 最優先 | オルタナを成長領域として見ているか / 顧客提案力の強化手段か / 自社運用能力を強化か / 外部マネージャー活用中心か。国内AMの方向性を見る上で重要 |
| A-8 | **Asset growth quality（受託残高成長の質）** | 最優先 | AUM絶対額ではなく: どの顧客層で伸びているか / 比率を維持したまま伸びているか / 外部顧客から増えているか / 市場上昇だけでなく純流入があるか |
| A-9 | **Fee resilience（料率低下圧力への耐性）** | 最優先 | 低コスト競争に巻き込まれやすい収益構造か / 高付加価値領域を持っているか / パッシブの規模勝負に寄るのか / オルタナ・ソリューション・機関投資家向け提案で収益の厚みを作れるか |

### B群: 運用の中身（最重要 — ここが差別化の本丸）

| # | 軸 | 優先度 | 見るべき観点 |
|---|---|:---:|------------|
| B-1 | **Alpha source（超過収益の源泉）** | 最優先 | その会社は超過収益をどこから得ると考えているか。情報の深さ / 企業変化の先読み / バリュエーションの歪み / 需給や行動バイアス / 業界構造の転換 / エンゲージメントによる企業価値改善 / ポートフォリオ構築の巧拙。**"何を見ているか"より"何が株価にまだ織り込まれていないと考えるか"** |
| B-1a | **└ Information edge（情報優位）** | 次点 | Alpha sourceの下位概念。何を情報優位の源泉とみなしているか / 調査への投資度合い / 定量・定性・データの接続 |
| B-1b | **└ Alternative data utilization（オルタナデータ活用度）** | 次点 | Information edgeの下位項目。衛星画像、POSデータ、特許、SNSセンチメント等。言及の有無だけでは差別化にならない — 何の仮説にどう使うかが重要 |
| B-2 | **Research architecture（調査体制・調査設計）** | 最優先 | PM主導か/アナリスト主導か / セクター制か/テーマ制か / クオンツが独立機能か/補助機能か / ESG/stewardshipが別部門か/運用に内包か / 外部情報の使用範囲 / 定量と定性の接続方法。**「クオンツがあるか」ではなく「クオンツをどこに差し込んでいるか」**。差はanalysis（分析）よりworkflow（意思決定フロー）への埋め込み方に出る |
| B-3 | **Decision rights（意思決定構造）** | 最優先 | FM個人裁量が強いか / 委員会型か / アナリストの提案権が強いか / エンゲージメント担当の意見が投資判断に入るか |
| B-4 | **Portfolio construction（ポートフォリオ構築）** | 最優先 | 集中投資か/分散か / benchmark-awareか/unconstrainedか / Active Share / Tracking Error / Turnover / 利確・損切り規律 / リスク配分の考え方 |
| B-5 | **Time horizon（投資時間軸）** | 最優先 | 四半期ベースか / 3〜5年で企業変化を待つか / 短期需給も重視するか / 長期の競争優位に張るか。**エンゲージメントを掲げていても時間軸が短ければ本当に企業変革まで付き合う運用ではない** |
| B-6 | **Risk management philosophy（リスク管理の思想）** | 最優先 | 下落局面への基本姿勢 / 損切り規律の強さ / ボラティリティと仮説崩れのどちらを重く見るか / リスク管理を守りと捉えるか超過収益創出の一部と捉えるか。個別ファンドの手法ではなく**会社としての運用観** |

### C群: スチュワードシップ

| # | 軸 | 優先度 | 見るべき観点 |
|---|---|:---:|------------|
| C-1 | **Stewardship integration（統合度）** | 最優先 | エンゲージメント内容がvaluation（企業価値評価）に反映されるか / 議決権行使がportfolio action（組入れ・縮小・売却）に結びつくか / engagementがIRイベントではなくinvestment thesis（投資仮説）の一部か。**「たくさん対話した」ではなく「対話が運用判断を変えたか」** |
| C-2 | **議決権行使方針** | 最優先 | 各社公開義務あり。行使方針の違いはAMの「色」が最も出る部分 |
| C-3 | **対話テーマ・実績** | 最優先 | どの企業に / 何を論点に / どのくらい継続し / 組入比率や投資判断がどう変わったか |
| C-4 | **Stewardship autonomy（独立判断性）** | 最優先 | 自前でどこまで判断しているか / ISS・Glass Lewisなどの議決権助言会社（proxy advisor）への依存度 / 議決権行使の説明がどこまで具体的か。**エンゲージメントの「本気度」の指標** |

### D群: 持続可能性

| # | 軸 | 優先度 | 見るべき観点 |
|---|---|:---:|------------|
| D-1 | **Economics/incentives（収益構造・インセンティブ）** | 最優先 | 手数料のどこで稼いでいるか / アクティブ運用にどこまでコストをかけられるか / 規模の経済が効くパッシブ寄りか / 人件費をかける調査寄りか。**その会社が本気でアクティブをやる経済合理性があるか** |
| D-2 | **Talent model（人材モデル）** | 最優先 | アナリスト→PM昇格ルート / 外部採用依存か内部育成か / 人材の専門性が業界・セクター単位か / クオンツ・データ人材の位置づけ。**"human judgmentをどこに残し、どこを機械化するか"** |
| D-2a | **└ Talent pipeline proxy（人材供給・育成の代理指標）** | 補助 | 新卒採用の継続性 / 中途採用比率 / 専門人材の確保方法 / 育成を語っているか即戦力依存か |
| D-3 | **親会社とのシナジー/制約** | 最優先 | 安定受託基盤がある反面、独立性が制約されるか / 親会社の意向で運用方針が歪むリスクはあるか |
| D-4 | **海外ネットワーク・リサーチ基盤** | 最優先 | 海外拠点の有無 / 外部委託比率 / グローバルリサーチへのアクセス |
| D-5 | **Organizational health（組織健全性）** | 補助 | 口コミ / 平均勤続年数 / 若手定着 / 組織文化の記述。**建前と実態のズレを見る補助材料**。主軸にはしない |

## 重みを下げてよいもの

以下は追加してもよいが主軸にはしない:

- オルタナティブデータ活用度の単独比較（B-1bとして下位に含める）
- 組織文化の細かい比較（D-5として補助に留める）
- 離職率そのものの比較
- 個別チーム単位の働き方差
- 「先進的な取り組みをしているか」という表面的な比較
