# スキル更新方針

skills は固定文書ではなく、実際の運用経験から継続的に育てる。

---

## 更新のトリガー

以下のいずれかが発生したらスキルを更新する:

1. **面接でのフィードバック**: 面接官に「わかりにくい」「答えになっていない」と言われた
2. **Claudeのレビュー見逃し**: Claudeが指摘しなかった問題点を後から発見した
3. **通過/落選パターン**: 特定の書き方をしたときの選考結果に傾向が見えた
4. **表現の改善**: より良い言い換え・パターンを発見した
5. **企業調査の失敗**: 面接で「その情報は古い」「その認識は違う」と言われた

---

## 更新方法

### Step 1: 気づきを記録
`docs/workflow/skill-update-log.md` に以下を記録する:

```
日付: YYYY-MM-DD
企業名: ○○
スキル名: [どのスキルに関係するか]
気づき: [何が問題だったか]
改善案: [スキルにどう反映するか]
```

### Step 2: スキルを更新

```
「[スキル名]スキルに[気づき]を反映して」
→ skill-creator スキルで更新
```

または直接 `.claude/skills/[スキル名]/SKILL.md` を編集する。

---

## 更新対象スキルの優先度

| 優先度 | スキル | 更新理由 |
|-------|-------|---------|
| 高 | es-writer | ES品質に直結 |
| 高 | readability-check | 面接官反応と直結 |
| 高 | company-fit-evaluation | 企業固有性に直結 |
| 中 | es-rewrite-patterns | 表現の改善に使う |
| 中 | interview-probe-generation | 面接対策の精度 |
| 低 | company-comparison-framework | 使用頻度が低い |

---

## 禁止事項

- 根拠のない「なんとなく追加」はしない
- 1つの更新で複数スキルを同時に大幅変更しない
- 更新後は既存のESで動作確認をする

---

## 更新履歴

更新内容は `docs/workflow/skill-update-log.md` に記録する。
