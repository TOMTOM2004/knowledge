# エージェント使用ガイド

各 subagent をいつ・どう使うかのガイド。

---

## 使い分けの原則

- **スキル（skill）**: 決まった手順で作業する（ES作成・調査・フォーマット変換等）
- **エージェント（agent）**: 特定の視点からレビュー・評価・質問生成を行う

スキルは「何を作るか」、エージェントは「どう評価するか」。

---

## ES系エージェント

### readability-reviewer
**使う場面**: ESを書き終えて「伝わりやすいか確認したい」とき

```
「[企業名].mdの可読性をチェックして」
「この設問回答が文系面接官に伝わるか見て」
```

**特徴**: 内容ではなく「伝わりやすさ」だけを評価。技術用語・長文・結論の遅さを指摘。

---

### company-fit-reviewer
**使う場面**: 「企業固有性が薄いかも」と感じるとき、提出前の最終確認

```
「[企業名].mdが[企業名]向けになっているか確認して」
「この志望動機はどこでも使えないか確認して」
```

**前提**: `company-info/[企業名]/research_brief.md` が存在すること

---

### role-fit-reviewer
**使う場面**: 職種系設問（なぜこの職種か・入社後何をしたいか）を書いたとき

```
「[企業名].mdが[職種名]向けになっているか確認して」
```

---

### question-fit-reviewer
**使う場面**: 書いたあとに「設問に答えているか不安」なとき（最優先で使う）

```
「[企業名].mdの各設問に正面から答えているか確認して」
「この設問ズレてないか厳しく見て」
```

**特徴**: 内容の良し悪しは評価しない。「聞かれたことに答えているか」だけを評価。

---

### consistency-overlap-reviewer
**使う場面**: 設問が3つ以上ある企業のES全体を見直すとき

```
「[企業名].mdの設問間の重複を確認して」
「全設問で話がかぶっていないか確認して」
```

---

### editor-refiner
**使う場面**: 他のreviewerの指摘を受けて実際に文章を改善したいとき

```
「question-fit-reviewerの指摘を踏まえて[企業名].mdを改善して」
「可読性の問題を修正して字数300字以内に収めて」
```

---

### research-gap-reviewer
**使う場面**: ES作成・面接準備前に調査品質を確認したいとき

```
「[企業名]の企業調査の不足を確認して」
「research_briefで足りていない論点を教えて」
```

---

### role-research-reviewer
**使う場面**: 職種理解が浅いと感じるとき、職種系設問の回答前

```
「[企業名]の[職種名]の職種調査に不足がないか確認して」
「会社理解と職種理解が混在していないか確認して」
```

---

### skeptical-interviewer
**使う場面**: ES完成後・面接準備時に「詰められそうな箇所」を把握したいとき

```
「[企業名].mdで面接で突っ込まれそうな質問を出して」
「この志望動機で2次面接で詰められる質問を生成して」
```

---

## エージェント選択チートシート

| 状況 | 使うエージェント |
|-----|---------------|
| 書き終えて最初に確認 | question-fit-reviewer |
| 「伝わるか」不安 | readability-reviewer |
| 「企業固有か」不安 | company-fit-reviewer |
| 「職種向けか」不安 | role-fit-reviewer |
| 3設問以上を総合確認 | consistency-overlap-reviewer |
| 指摘を受けて文章直す | editor-refiner |
| 提出前の一括チェック | es-review-protocol スキル |
| 企業調査が薄い | research-gap-reviewer |
| 職種調査が薄い | role-research-reviewer |
| 面接前に自己チェック | skeptical-interviewer |
