# 品質ゲート定義

軽量化しても品質を落とさないための最終確認ルール。

---

## 原則

- 「毎回全部チェック」ではなく「段階ごとに必要なものだけチェック」
- script で確認できるものは先に script で済ませる
- agent は意味理解が必要な判定のみに使う
- 最終品質ゲートは必ず通す

---

## 段階別 品質チェック表

### Gate 1: 機械チェック（提出前必須）

```bash
python tools/checks/es_checker.py 移行後ES/<企業名>.md
```

| チェック項目 | 合格基準 |
|------------|--------|
| 文字数 | 設問別の指定範囲内 |
| 禁止表現 | 0件 |
| 企業名表記 | 統一されている |
| 設問未回答 | 0件 |
| 重複表現 | 同一フレーズ 3回未満 |

**ERROR があれば先に修正。WARNING は判断して対処。**

---

### Gate 2: 設問・可読性確認（草案後・必須）

| reviewer | 確認内容 | 合格基準 |
|---------|---------|--------|
| `question-fit-reviewer` | 設問に正面から答えているか | 全設問で「答えている」判定 |
| `readability-reviewer` | 文系若手面接官に伝わるか | 専門用語・抽象語の指摘なし |

**依頼文**:
```
「[企業名].md の設問適合と可読性を確認して」
```

---

### Gate 3: 企業・職種適合確認（仕上げ前・必須）

| reviewer | 確認内容 | 合格基準 |
|---------|---------|--------|
| `company-fit-reviewer` | 企業固有性があるか | 一般論「どこでも言える」がない |
| `role-fit-reviewer` | 職種適合しているか | 職種業務・キャリアパスへの接続あり |

**前提**: `company-info/<企業名>/research_brief.md` が存在すること

**依頼文**:
```
「[企業名].md の企業適合と職種適合を確認して。
 company-fit-reviewer と role-fit-reviewer を使って」
```

---

### Gate 4: 重複・一貫性確認（3設問以上の場合・必須）

| reviewer | 確認内容 | 合格基準 |
|---------|---------|--------|
| `consistency-overlap-reviewer` | 設問間のエピソード・強みの重複 | 重複エピソードなし、強みが設問で重ならない |

**依頼文**:
```
「[企業名].md の設問間の重複を確認して」
```

---

### Gate 5: 面接深掘り耐性（面接前・任意）

| reviewer | 確認内容 | タイミング |
|---------|---------|---------|
| `skeptical-interviewer` | 詰められそうな弱点 | 面接 1週間前以降 |

**依頼文**:
```
「[企業名].md で面接で突っ込まれそうな質問を出して」
```

---

### Gate 6: 文面改善（任意）

| reviewer | 確認内容 | タイミング |
|---------|---------|---------|
| `editor-refiner` | 具体的な改善文案の生成 | 指摘を受けた後 |

---

## 時間がない時の最小確認セット

Gate 1 + Gate 2 + Gate 3 を通す:

```bash
# Step 1: 機械チェック
python tools/checks/es_checker.py 移行後ES/<企業名>.md

# Step 2: 設問適合 + 可読性
「[企業名].md の設問適合と可読性を確認して」

# Step 3: 企業・職種適合
「[企業名].md の企業適合と職種適合を確認して」
```

---

## 重要企業向けの拡張確認

第一志望・最終前提出の場合は全 Gate を通す:

```
Gate 1 → Gate 2 → Gate 3 → Gate 4 → Gate 5 → Gate 6
```

---

## 通す Gate を省略してよい条件

| Gate | 省略できる条件 |
|-----|-------------|
| Gate 4（重複） | 設問が 2つ以下 |
| Gate 5（面接） | 書類選考段階 |
| Gate 6（改善） | 指摘がなかった場合 |

---

## 企業固有性・職種適合・設問適合の判定基準

### 企業固有性の合格基準
- その企業の固有名詞・数値・戦略が含まれている
- 競合他社名を入れても違和感ない表現が含まれていない
- research_brief の「企業固有性」から少なくとも 2点が反映されている

### 職種適合の合格基準
- 職種の業務内容に触れている
- 「なぜこの職種か」が論理的に説明されている
- スキル・経験との接続が具体的

### 設問適合の合格基準
- 設問で問われていることに直接答えている（最初の 2文で）
- 問われていないことを長く書いていない
- 設問の制約（〇〇を踏まえて等）を守っている
