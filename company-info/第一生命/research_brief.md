### 軸2: 配属部署・コースの具体業務

#### ■ 資産運用コース

**運用資産規模：約70兆円**（国内機関投資家トップクラス）

**フロント部門：**
| 部署 | 主な業務 |
|---|---|
| 株式部 | 国内株式・外国株式の銘柄選定・ポートフォリオ管理 |
| 債券部 | 外国債券（新興国含む35カ国・23通貨） |
| オルタナティブ投資部 | インフラ・物流施設・航空機ファイナンス等の投融資 |
| 特別勘定運用部 | 変額保険等の分離勘定運用 |

**ミドル・バック部門：**
- 運用企画部（ALM・資産配分戦略策定）
- 総合審査部（与信・リスク審査）
- 運用資産管理部・運用サービス部

**アウトプット例：** テーマレポート・ポートフォリオ提案メモ・ESGエンゲージメント記録・議決権行使判断

**資産運用の構造（自己勘定 vs グループ運用会社）：**
- 一般勘定（自己勘定）＝第一生命本体が直接運用。保険負債とのALMが前提
- 特別勘定＝AM One（アセットマネジメントOne）等に投資顧問委託
- AM One：みずほFGと第一生命の合弁（みずほ70%・第一生命30%）。顧客向けファンドが主体
- 資産運用コースで担うのは主に**一般勘定（自己勘定）の直接運用**

**運用ポートフォリオの転換方向（2024〜2030年）：**
- 国内株式：2024〜2026年度で1.2兆円削減計画、2030年度末に最大1.5兆円まで圧縮
- オルタナティブ：現在3兆円超（インフラ・PE・不動産等）から引き続き拡大
- 公社債中心のポートフォリオを維持しつつ、選別的クレジット・インフラへの投融資を積極化

#### ■ ITデジタルコース

**【最重要】組織構造の理解：**

| 組織 | 役割 | 本体採用との関係 |
|---|---|---|
| 第一生命本体「IT企画部」 | 戦略企画・要件定義・ユーザー部門との調整・ベンダーガバナンス・IT投資判断 | **ITデジタルコース採用者の主配属先** |
| 第一生命テクノクロス（DLTX） | 開発・運用・テスト・インフラ管理・DX人財育成・インド拠点 | 本体IT企画部と一体化・出向協業あり |

**本体採用（ITデジタルコース）で入る意味：**
- IT企画部でビジネス側（保険・運用・営業）と技術側の橋渡し役を担う「事業IT人材」として育成される
- 開発の実作業はテクノクロスが担い、本体社員はユーザー部門との要件整理・プロジェクト管理・IT戦略立案が中心
- テクノクロス採用（別途新卒採用あり）とは立ち位置が根本的に異なる

**テクノクロスの位置づけと変化（2024年）：**
- 旧社名：第一生命情報システム → 2024年4月「第一生命テクノクロス」に改称
- 同時に「第一生命保険の子会社」→「第一生命HDの直接子会社」へ格上げ
- 役割変化：「第一生命のシステムを守る会社」→「グループ全体のIT・DX推進をリードする会社」
- テクノクロスの弱点（OpenWork口コミ）：外販事業が弱く、基本的に第一生命グループのみ。親会社の予算に依存する構造は残る

- DX推進担当：新規AI/データ施策が中心（変革7割・保守3割程度）
- 本体採用のITデジタルコースはDX推進寄りのプロジェクト関与比率が高い

- 「アジャイルとウォーターフォールが共存する現場」が実態

**AI活用の実態（2024〜2026年）：**

|---|---|---|

### 軸4: 志望コース vs 他コースの差別化軸

#### 資産運用コース を選ぶ積極的な理由
- 運用資産70兆円という国内最大規模の機関投資家として、個人投資家にはアクセスできない資産クラス（インフラ・オルタナティブ・新興国債券）で意思決定できる
- 「受託者責任（Fiduciary Duty）」という高い倫理水準のもとで運用する環境
- 資産運用立国・ESG投資の最前線という社会的意義
- アクチュアリーコース（数理保険料算出が中心）・クオンツコース（モデル構築が中心）との違いは「ポートフォリオ全体の投資判断とエンゲージメント」という点

**「なぜ資産運用コースか」設問への角度：**
「70兆円という規模でしか起こせない長期インパクト（エンゲージメント・日本株底上げ）に、受託者責任という責任の重さが伴う環境で挑戦したい」

#### ITデジタルコース を選ぶ積極的な理由
- 「内製化」を中計に明記した大手生保は希少。自ら企画した仕組みが全国1,100拠点の営業現場に即実装されるスケール感
- 本体のIT企画部配属により、エンジニアではなく「事業視点でITを企画・推進する人材」として育つ環境
- 保険コアシステム（数十年単位のデータ）×データ基盤（DMAP）×AIという多層的な技術課題が共存する設計環境
- 上場会社の資本効率要求がIT投資判断にも直結（投資対効果を数字で説明する文化）
- 非保険（ベネフィット・ワン）まで含むウェルネスエコシステム全体のデジタル化で、他の大手生保よりスコープが広い

**「なぜITデジタルコースか」設問への角度：**
「自動化・AI活用を設計する側として価値を出したいが、生保ドメインを深く理解しないと要件定義の質は担保できない。70兆円の運用基盤と1,100拠点の営業現場を動かすIT設計が最も試し甲斐のある挑戦」

**7-5. 「なぜ事業会社ITか」6段反論軸（ITデジタルコース向け）**

面接で詰められた際に6段で切り分けて答える。順に整理する。

```
① なぜITに関わりたいのか（IT全般の理由）
   自動化・仕組み化による「再現性ある改善」が最も価値を生む仕事だと考えるから。
   人が毎回判断するより、適切な設計が一度機能すれば無限に価値を出し続ける。

② なぜSIer・ITコンサルではないのか（外販IT vs 事業会社IT）
   SIerは「作る」ことが目的化しやすく、運用・定着が別主体に委ねられる。
   事業会社のIT部門は、設計から定着・効果測定まで一気通貫で担える。
   自分は「使われること」を最終目的とするため、事業会社内のIT企画が合う。

③ なぜ保険業界のITなのか
   保険は長期契約・複雑商品・高い規制要求という条件が重なり、
   IT設計の難易度と社会的インパクトが最も高い業界の一つ。
   難しい制約の中でどう価値を出すかに知的面白さを感じる。

④ なぜ損保ではなく生保なのか
   生保は保険期間が数十年に及び、契約1件1件のデータが超長期的価値を持つ。
   このデータ資産を活用したALM設計や顧客理解の深さは損保とは別次元。

⑤ なぜ第一生命なのか（競合差別化）
   IT内製化を中計に明示した大手生保。上場会社ゆえのROI要求がIT投資判断にも及び、
   数字で価値を語る文化が自分の強みと合致する。
   ウェルネスエコシステム（ベネフィット・ワン）まで含むスコープの広さも魅力。

⑥ なぜITデジタルコースなのか（資産運用コースではなく）
   保険営業現場や資産運用基盤を、ITの設計者として「ビジネス側から最も近い位置で」
   変革できる役割が欲しい。コードを書くより「何を・なぜ作るか」を設計する
   IT企画の立場が、自分のアーキテクト思考と一致している。
```

**7-6. 相互会社 vs 上場会社：IT投資判断の構造的違い**

面接で「なぜ日本生命ではなく第一生命か」を深掘りされた際、組織形態の違いをIT投資判断の文化差として語れると格段に深みが増す。

| 観点 | 相互会社（日本生命・明治安田・住友生命） | 上場会社（第一生命） |
|---|---|---|
| IT投資の正当化 | 社員総代会への説明は形式的。投資対効果の外部公開義務なし | 株主・アナリストへのIR開示義務あり。IT投資のROIを数字で語る文化が根付く |
| 長期投資のしやすさ | 株主配当不要→剰余金を内部留保・長期IT投資に充てやすい | 四半期利益への圧力あり→短期効果も問われる |
| 資金調達力 | 資本市場からの調達不可（株式発行なし）→大型IT投資時の資金制約リスク | 増資・社債等で資金調達可能→大規模IT投資の財務余力 |
| 意思決定スピード | 理論的には株主圧力なく長期視点で可。ただし社内合意重視で稟議が遅い傾向 | 資本効率要求から「投資対効果が見えない案件」は削減されやすい→逆にROI明確な案件は通りやすい |
| 人材確保競争力 | 非上場のため株式報酬（RSU・ストックオプション）が使えない | 上場企業として株式報酬を活用したIT人材獲得が可能 |

**面接での使い方（ITデジタルコース志望）：**
```
「相互会社3社はIT投資の説明可能性が外部に問われる仕組みが薄く、
DX投資の効果を数字で示す文化が育ちにくい面があります。
第一生命はROEや投資対効果をIRで問われる上場会社として、
IT施策の価値を定量的に説明する文化が根付いています。
自分がデータ×ITで価値を出すなら、その価値を正当に評価してもらえる環境が重要で、
第一生命が最も合っていると判断しました。」
```

⚠️ **注意**：相互会社を批判的に語りすぎると減点。「文化の違い」として中立に述べた上で、自分が第一生命を選ぶ理由を積極的に語る形に留める。

**【NGパターンの指摘】**
- ❌「大手だから安心・安定」→ 日本生命・明治安田も同様。差別化にならない
- ❌「幅広い仕事ができる」→ スペシャリティコースを選ぶ理由として矛盾
- ❌「海外展開していてグローバルに働ける」→ 資産運用コースは基本東京勤務。海外赴任はキャリア後半
- ❌「内製化を進めているのが魅力」→ 理由が浅い。なぜ内製化が重要か・自分がどう関わるかまで言わないと空虚
- ❌「DXを進めているので貢献したい」→ これだけでは競合でも言える。具体のシステム・課題・自分の接続まで語る

### C. ITデジタルコースでやりたいことの優先順位

**主軸（面接で最初に語る）：AI定着設計**
- 根拠：「作ることより使われること」という自分の仕事哲学と最も直結
- 具体：デジタルバディ（2026年度全国展開）・生成AI基盤の現場定着。ユーザー部門の業務理解×IT設計で「現場が実際に使う仕組み」を作る
- 語り方：「AIを導入して終わりではなく、現場が日常的に使い続ける状態をゴールに定義し、運用設計まで担いたい」

**第二軸（専門性の証明として）：データ基盤（DMAP）**
- 根拠：DS経験・クラウドデータ基盤整備経験と技術的親和性が高い
- 具体：データ定義統一・品質向上・AIへのデータ供給設計（どんなデータをどの粒度・形式でAIに渡すか）
- 語り方：「AIの精度は結局データ品質に依存する。データ基盤を整備することがAI活用全体の底上げになる」

**第三軸（インパクト可視化として）：営業支援（DL PadⅢ改善）**
- 根拠：全国1,100拠点という具体的な受益者が明確で、成果が数字で見えやすい
- 具体：提案シミュレーション精度向上・成約率・顧客満足度スコアへの貢献
- 使い所：「成果が数字として見えることを重視している」という自己PR設問と連動させる

**意図的に避ける軸（初期段階）：保険コアシステム**
- 理由：技術難易度と組織的な制約が高く、若手が早期に変革を起こせる余地が限られる。深みはあるが初期のやりたいこととして語るには実態と乖離しやすい

### E. 競合比較の一問一答

**「日本生命ではなく第一生命は？」（IT・資産運用 共通）**
```
最も大きな違いは、組織形態と資本規律の違いが生み出す「数字で語る文化」です。
日本生命は相互会社のため株主への説明責任がなく、IT投資・運用の費用対効果を
外部に説明する義務がない。第一生命は上場会社として株主・アナリストに
IR開示する義務があり、「何のためにこの投資をするのか」を定量的に語る文化が
根付いています。
また、日本生命はIT体制でTIS・インテックとの外部提携を強化する一方、
第一生命は内製化を中計に明示し、自社人材でDXを推進する姿勢が際立ちます。
この環境で、ITの価値を自ら設計して数字で証明できる人材として成長したい。
```

**「日本生命ではなく第一生命は？」（資産運用コース向け補足）**
```
運用面では、日本生命は運用資産約85兆円と規模で上回りますが、
相互会社として四半期利益への外部圧力がない分、
リスク管理の説明可能性を外部に問われる緊張感は薄い。
第一生命は上場会社として「何に、なぜ投資するか」を
IRで問われる環境があり、投資判断の説明可能性を最優先とする
自分の投資哲学（説明可能性>下落耐性>検証可能性）と合致します。
```

IT内製化を中計に明記。設計から現場定着まで自社主導で担う体制は
