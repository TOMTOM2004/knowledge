# 銀行・証券・信託 — 三者連携の構造ガイド

_2026-04-20 作成 / Career Lounge SC面談前の基礎理解用_

---

## 1. そもそも3者は何をしているのか

### 銀行（みずほ銀行）— 「貸す」「預かる」「送る」

企業や個人から預金を集め、そのお金を企業に貸し出す（融資）。決済（振込・送金）も担う。

- **法人向け**: 融資（運転資金・設備投資）、決済、為替、コミットメントライン
- **個人向け**: 預金、住宅ローン、カードローン
- **RM（リレーションシップマネージャー）**: 法人の窓口。経営者と直接対話して課題を発見し、グループ全体のソリューションに繋ぐ司令塔

> **一言**: 顧客との最も近い接点。顧客の経営課題やライフイベントを最初に把握する立場

### 証券（みずほ証券）— 「集める」「売買する」「助言する」

企業が株式や社債を発行して投資家から資金を調達するのを仲介する。投資家には運用商品を提供する。M&Aの助言も行う。

- **DCM（社債引受）**: 企業が社債を発行する際の引受・販売。みずほは12年連続首位
- **ECM（株式引受）**: IPO（新規上場）や増資の引受・販売
- **M&Aアドバイザリー**: 企業の合併・買収の助言。Greenhill買収で海外M&A能力を内製化
- **GM（グローバルマーケッツ）**: 株式・債券・為替のトレーディング
- **リテール**: 個人投資家への投資信託・株式の販売
- **リサーチ**: 企業・産業の分析レポートを投資家に提供

> **一言**: 資本市場（株・債券）と顧客を繋ぐ。「借りる」以外の資金調達手段を提供する

### 信託（みずほ信託銀行）— 「守る」「管理する」「承継する」

財産を「預かって、本人の代わりに管理・運用・処分する」のが信託の本質。銀行が「お金を貸す」のに対し、信託は「財産を預かって守る・次世代に渡す」。

- **事業承継信託**: オーナー経営者の自社株を信託に移し、後継者への承継を計画的に実行
- **遺言信託**: 遺言書の作成・保管・執行を受託
- **不動産信託**: 企業の保有不動産を信託し、流動化・証券化・管理を行う
- **年金信託**: 企業年金の資産を受託し、運用・給付を管理
- **証券代行**: 上場企業の株主名簿管理、配当金支払い、株主総会の事務代行

> **一言**: 「今ある財産を守り、次世代に渡す」専門家。特に事業承継で不可欠

## 3. 銀行 ⇔ 証券 — 「借りる」から「市場で調達する」へ

### 連携がない場合のデメリット（機会損失）

企業が成長し、銀行融資だけでは資金調達の選択肢が足りなくなる場面がある。

- 社債を発行したい → 銀行では扱えない。別の証券会社を探す必要がある
- M&Aで会社を買いたい → 銀行は融資はできるがM&Aの助言はできない。別のアドバイザーを探す
- IPOしたい → 銀行では対応不可。証券会社に一から関係構築

**問題**: 証券会社に一から説明し直し。銀行が把握している経営課題・財務状況・経営者の本音が証券側に伝わらない。提案の質が下がり、スピードも落ちる。

### 連携がある場合のメリット

```
【企業が社債を発行したいケース】

連携なし:
  企業 → 銀行RM「社債出したい」→「うちではできません。○○証券を紹介します」
       → ○○証券「初めまして。御社の事業について教えてください」←ゼロから説明
       → 数週間かけて企業理解 → 提案 → 発行

連携あり（みずほ）:
  企業 → 銀行RM「社債出したい」→ RM が証券DCMチームに連携
       → 証券側はRMから企業情報・財務状況・経営方針を共有済み
       → 即座に最適な条件設計 → 提案 → 発行
       → 融資残高の一部を社債にリファイナンス → 企業は調達手段を分散できる
```

### 具体的な連携シナリオ

| きっかけ（銀行RMが発見） | 証券が提供するソリューション | 銀行へのリターン |
|------------------------|--------------------------|----------------|
| 融資先が成長して調達を多様化したい | 社債発行（DCM） | メイン取引の維持・手数料 |
| 融資先がM&Aで事業拡大したい | M&Aアドバイザリー | 買収資金の融資（LBOファイナンス等） |
| 融資先がIPOを目指している | IPO主幹事・引受 | 上場後のメインバンク取引の継続 |
| 融資先の資金繰りが改善し余資が発生 | 運用商品の提供 | 顧客満足度向上・取引深耕 |

| きっかけ（証券が発見） | 銀行が提供するソリューション | 証券へのリターン |
|----------------------|--------------------------|----------------|
| M&A案件で買収資金が必要 | LBOファイナンス・融資 | 案件成立で手数料確定 |
| IPO企業が上場後に設備投資したい | 成長融資 | 上場後もグループ内で継続取引 |
| リサーチで業界動向を把握 | RMが顧客に情報提供 → 信頼構築 | RMが証券案件のニーズを発見する確率UP |

## 4. 銀行 ⇔ 信託 — 「経営者の事業」と「経営者の家族・資産」

### 連携がない場合のデメリット

中堅中小企業のオーナー経営者にとって、「会社の経営」と「個人の資産」は不可分。

- 事業承継を考えたい → 銀行は融資面のサポートはできるが、自社株の信託・遺言の執行は専門外。別の信託銀行を探す
- 会社の不動産を活用したい → 銀行は融資担保としては見ているが、不動産の流動化・証券化は信託の領域
- 年金制度を設計したい → 銀行では対応不可

**問題**: 経営者は「会社のこと」と「家族のこと」を別々の金融機関に相談する。情報が分断され、最適な設計ができない。例えば、自社株の承継方法と融資条件は連動するのに、別々に検討されてしまう。

### 連携がある場合のメリット

```
【事業承継のケース — 最も銀信連携が活きる場面】

連携なし:
  オーナー社長（70歳）→ 銀行RM「後継者に会社を渡したい」
  → 銀行「融資の継続は大丈夫です」（しかし自社株の移し方は分からない）
  → 社長が別途、信託銀行や税理士を探す
  → 信託側は会社の財務状況や融資条件を知らない
  → 自社株評価と融資条件がちぐはぐになるリスク

連携あり（みずほ）:
  オーナー社長 → 銀行RM「後継者に会社を渡したい」
  → RMがみずほ信託の事業承継チームに連携
  → 信託側はRMから会社の財務状況・融資条件・経営者一族の関係を共有済み
  → 自社株の信託設計 × 融資条件の再設計 × 遺言信託を一体で提案
  → さらにM&Aを選ぶなら証券のM&Aチームも参加 → 3者一体の提案
```

### 具体的な連携シナリオ

| きっかけ（銀行RMが発見） | 信託が提供するソリューション | 銀行へのリターン |
|------------------------|--------------------------|----------------|
| オーナー社長が後継者問題を抱えている | 事業承継信託（自社株を信託に移管し計画的に承継） | 承継後も融資関係を維持。後継者との新たなRM関係構築 |
| 融資先企業が本社ビルを保有している | 不動産信託（流動化・証券化で資金化） | 資金化した分で融資の借り換え or 新規投資 |
| 経営者が個人資産の管理を相談 | 遺言信託・資産管理信託 | 個人の信頼構築 → 法人取引の深耕に繋がる |
| 融資先が企業年金を導入したい | 年金信託（掛金の受託・運用・給付管理） | 企業の福利厚生ニーズに応え取引関係を強化 |

| きっかけ（信託が発見） | 銀行が提供するソリューション | 信託へのリターン |
|----------------------|--------------------------|----------------|
| 事業承継で後継者が設備投資したい | 成長融資・運転資金融資 | 承継案件の実行が円滑に進む |
| 不動産信託で新規開発が決まった | 不動産プロジェクトファイナンス | 不動産案件の規模拡大 |

## 5. 証券 ⇔ 信託 — 「資本市場」と「資産管理」

### 連携がない場合のデメリット

- IPO後の企業 → 株主名簿管理（証券代行）が必要だが、IPO主幹事と別の信託銀行に依頼する
- M&Aで会社を売却 → 売却資金の管理・運用・承継プランが未設計のまま
- 年金の運用 → 信託が受託するが、運用商品は別の証券会社から調達する

### 連携がある場合のメリット

| きっかけ（証券が発見） | 信託が提供するソリューション |
|----------------------|--------------------------|
| IPO企業が上場する | 証券代行（株主名簿管理・配当金支払い・株主総会事務） |
| M&Aで企業を売却したオーナー | 売却益の資産管理信託・遺言信託 |
| 機関投資家が年金を運用したい | 年金信託の受託 + 証券のトレーディングで執行 |

| きっかけ（信託が発見） | 証券が提供するソリューション |
|----------------------|--------------------------|
| 事業承継でM&Aを選択 | M&Aアドバイザリー（バリュエーション・交渉・実行） |
| 不動産信託の証券化 | 不動産証券（REIT・CMBS等）の引受・販売 |
| 年金資産の運用 | 投資信託・仕組債等の運用商品提供 |

## 6. 三者一体のフルシナリオ — 事業承継

> 銀証信の連携価値が最も明確に現れるのが事業承継。1つの案件に3者全てが関わる。

```
 【ステージ1: 課題発見】
   銀行RM が融資先のオーナー社長（70歳）と面談
   → 「後継者がいない。会社をどうするか悩んでいる」
   → RM は経営者の本音・家族関係・会社の財務を知っている

         ↓ グループ内で情報共有（オプトアウト方式）

 【ステージ2: 選択肢の提示 — 3者が同時に動く】

   ┌─────────────────────────────────────────────────────┐
   │                                                       │
   │  選択肢A: 親族内承継                                    │
   │  ├── 信託: 自社株の信託移管 + 遺言信託で承継設計         │
   │  ├── 銀行: 後継者への融資条件の再設計（保証人変更等）    │
   │  └── 証券: 不要（この選択肢では出番なし）               │
   │                                                       │
   │  選択肢B: M&Aで第三者に売却                             │
   │  ├── 証券: M&Aアドバイザリー（買い手探し・バリュエーション・交渉）│
   │  ├── 銀行: 買い手への買収資金融資（LBOファイナンス）     │
   │  └── 信託: 売却益の資産管理信託 + オーナーの遺言信託     │
   │                                                       │
   │  選択肢C: MBO（経営陣による買収）                       │
   │  ├── 証券: MBOのスキーム設計・バリュエーション            │
   │  ├── 銀行: MBO資金の融資                                │
   │  └── 信託: 自社株の信託管理 + オーナーの退職後資産管理    │
   │                                                       │
   └─────────────────────────────────────────────────────┘

         ↓ どの選択肢でも3者が連携して実行

 【ステージ3: 実行後のフォロー】

   銀行: 承継後の後継者 or 新オーナーとの取引継続
   証券: 成長した企業のIPO・社債発行（将来の案件の種）
   信託: 旧オーナーの資産管理・年金・相続（生涯の顧客化）
```

## 7. 連携がないことの損失を可視化する

### 定量的な損失

| 場面 | 連携なし | 連携あり | 差分 |
|------|---------|---------|------|
| 社債発行の提案機会 | 銀行RMが「うちの仕事じゃない」で見過ごす | RMが証券DCMに繋ぐ → 引受手数料 | 年間数億～数十億円のグループ収益機会損失 |
| 事業承継のフルパッケージ | 銀行は融資継続のみ。信託・証券の手数料は他社に流出 | 3者一体で提案 → 手数料をグループ内で獲得 | 1件あたり数千万～数億円の差 |
| IPO後の継続取引 | IPO主幹事の証券と、メインバンクが別グループ → 関係が分散 | IPO→メインバンク→証券代行まで一気通貫 | 上場後10-20年の取引関係の差 |

### 定性的な損失

| 場面 | 連携なし | 連携あり |
|------|---------|---------|
| 顧客体験 | 同じ話を3回する。担当者が変わるたびに関係構築やり直し | 1人のRMが窓口。背景を理解した専門家が即座に参加 |
| 提案の質 | 融資・証券・信託がバラバラに提案 → 整合性がない | 3者が連携して最適な組み合わせを設計 |
| スピード | 別の金融機関に紹介 → 審査・与信やり直し → 数週間～数ヶ月のロス | グループ内で情報共有済み → 即座に動ける |
| 経営者の本音 | 会社の話は銀行に、家族の話は信託に、市場の話は証券に → 分断 | 1つのグループが全てを把握 → 本質的な課題に到達できる |

## 8. テクノロジーによる連携強化の方向性

### 現在（2026年時点）

| 連携インフラ | 内容 |
|------------|------|
| かなで + WITH | 人事・評価の統一 → 人の異動が円滑に |
| オプトアウト | 法人顧客情報の銀行→証券共有を顧客同意なしで可能に（拒否しない限り） |
| プラネットブース | 銀行の店舗内に証券社員が常駐 → 物理的なワンストップ |
| Wiz Chat / Wiz Base | グループ共通のAI基盤 → 情報検索の統一 |

### テクノロジーで強化できる領域（仮説）

| 課題 | テクノロジーによる解決 |
|------|---------------------|
| RMが証券案件の種を見逃す | CRMにAIが「この融資先は社債発行の適性あり」とアラート → RMが証券に繋ぐ |
| 産調レポートが現場に届かない | AIが担当先の業種に合わせてレポートを自動フィルタリング・要約してRMに配信 |
| 事業承継の最適スキーム設計に時間がかかる | AIが企業の財務・株主構成・家族関係から承継シナリオを自動シミュレーション |
| UPSIDER決済データの活用 | リアルタイム決済データから資金繰り予測 → RMが先回りして提案 |
| 顧客情報のグループ横断検索 | Wiz Baseが銀行・証券・信託の顧客データを統合検索 → 重複提案の防止・隠れたニーズの発見 |

## 9. SMFGに信託がないことの具体的な影響

> 「銀証信一体は他メガもやっている」への反論として最も有効

**SMFGの構造的弱点**: 三井住友信託銀行は完全に別グループ（三井住友トラストHD、東証上場）。SMFGからは切り離されており、情報共有・送客・人事交流は**外部企業間の連携**と同じレベルのハードル。

```
みずほFG:  銀行 ←→ 証券 ←→ 信託   ← 全て100%グループ内
MUFG:     銀行 ←→ 証券(合弁60:40) ←→ 信託   ← 証券に外資の制約
SMFG:     銀行 ←→ 証券 ←→ 信託なし ← 信託は完全別グループ
```

**影響**: 事業承継でM&Aを選択 → SMBCは証券がM&Aを助言できるが、売却後のオーナーの資産管理・遺言信託はグループ内で提供できない。顧客は別の信託銀行を探す必要があり、ワンストップにならない。

   │  経営課題を発見    │  │  調達手段を提供    │
    ──────────────────    ──────────────────
          │      ＼             ／     │
          │       ＼           ／      │
          │        ＼         ／       │
          │     ──────────────────     │
          │    │     信  託        │    │
          │    │  事業承継・資産管理│    │
          └────│  【財産を守る橋】 │────┘
