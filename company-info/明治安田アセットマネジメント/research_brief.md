## 軸1: 事業・ビジネスモデル

**事業概要**
- 明治安田生命保険100%子会社。1986年設立。従業員219名（少数精鋭）
- 売上高 約110億円（2024年3月期）
- アクティブ運用を中心とした「チームアプローチ型」の運用哲学が特徴
- 主要サービス：公募投資信託、年金・機関投資家向け一任運用、グループ内受託

**注力成長領域**
- **プライベートアセット**：中計3ヵ年で6,000億円を海外プライベート資産に投資（親会社主導）
- **Man Group（英国）提携**（2025年2月）：10年間で1,500億円をプライベートデット中心に委託
- **インパクトファイナンス**：FY2024実績840億円、中計目標を1,700億円に上方修正
- **ESG投資**：NextGen ESG Japan Fundへ50億円シード出資、EU SFDR9条準拠

### IT職

- **グループIT会社の有無**：**明治安田システム・テクノロジー株式会社（MYTECNO）** が明治安田グループのIT専門会社として存在。主に明治安田生命の基幹システム（新契約・契約管理・支払・営業管理）を担当するSIer。MYAM本体のIT職はクオンツAI・運用データ基盤等の「アセマネ固有の」システムを担当すると推測される（詳細は**【要確認】**）
- **内製化状況（グループ全体）**：親会社・明治安田生命が2025年1月にアクセンチュアと5年間DXパートナーシップを締結（約300億円投資）。2025年4月から「デジタルイノベーションHub（100人体制）」を本格稼働。内製化人材300名育成を目標に設定。ただしこれはMYAMではなく明治安田生命本体の施策。MYAMへの波及度は**【要確認】**
- **AI活用実態**：運用企画部がクオンツ・AIモデルを「独自開発」と公式表明 → 本番適用段階の可能性あり。社長インタビュー（2024年）でもアクティブ運用強化の文脈でAI活用に言及。PoC段階か本番定着かは**【要確認】**
- **ユーザー部門との距離**：少数精鋭（219名）のため、IT担当者はビジネス部門（運用・企画）と近い距離で働く可能性が高い。OpenWork口コミ「中途入社者・出向者が多い」ため、本体IT職の立ち位置はビジネス寄りと推測
- **「変革」vs「保守」**：親会社グループ全体のDXは加速中だが、MYAM本体の口コミでは「保守的・官僚的風土」「プロ意識の希薄さ」の指摘が複数あり、変革推進には社内調整が伴う

## 軸4: 志望コース vs 他コースの差別化軸

### 運用職を選ぶ積極的理由
- **インサイトが直接投資判断に反映される**最前線。「分析→意思決定」の最短経路
- アクティブ運用 = 「なぜこの銘柄か」の言語化が毎回必要 → 複数シナリオ設計力が直接活きる
- 超長期・生保ALMという独自の視座で運用できる（他AMにない強み）
- 10年後にFMを目指すキャリアゴールと完全一致

### 企画・IT職を選ぶ積極的理由
- 「型化→組織資産化」という強みが最大限発揮できる
- 運用の高度化（クオンツ・AI・商品開発）という会社の最重要課題に直接関われる
- IT職：少数精鋭ゆえ「AIへのデータ提供設計」というアーキテクト志向が活きる環境
- 運用職へのキャリアチェンジも視野に入れつつ、まず会社の仕組みを設計する側に立てる

**「なぜ運用職でなく企画/IT職か」の説明が必要な場合**：「運用の前段階である仕組みと基盤を整えることで、組織全体の意思決定品質を底上げしたい。個別銘柄の分析より、組織が再現可能な判断をできる構造を作ることが自分の強みと合致する」

## 軸6: 企業の強み・弱み（絶対評価）

**強み**
- **安定受託基盤**：親会社45兆円の安定資金 → 業績悪化リスクが低く長期視点で仕事できる
- **PRI評価4項目5つ星（2024年次評価）**：PRIの最高評価を複数項目で獲得（出典: stewardshipreview_2025.pdf）。ただしPRI署名自体は2016年3月で日本最古参（ニッセイAM 2006年）ではない点に注意
- **少数精鋭**：219名。若手から裁量ある仕事が可能。意思決定が速い
- **Man Group提携**：グローバルなプライベートアセット知見を取り込める

**弱み・課題（貢献余地として）**
- **AUM規模が業界中小クラス**（TOP10圏外） → 外部顧客・公募投信の拡大が課題。商品開発・マーケの強化余地大
- **保守的・官僚的体質**（OpenWork口コミ複数）：「保守的体質」「プロ意識の希薄さ」「切迫感の欠如」が複数指摘。親会社（生保）色が強く意思決定が遅い → DX・AI活用加速を担うIT/企画職での貢献余地大
- **成長性の限界**（OpenWork口コミ）：「成長性はほぼない」「年功序列」「上が詰まっている」という評価あり。中途入社者・出向者が多い組織構成 → 新卒・若手の視点から変革提案をする余地が逆に存在
- **知名度不足** → ブランディング・情報発信の強化余地
- **採用人数の少なさ**（数名程度） → 少人数ゆえ1人が担う業務範囲が広く、成長機会が多い反面、即戦力化が求められる

**OpenWork評価（参考）**
- 総合評価: 2.75/5（23名回答）
- 残業: 月平均24時間
- 有給消化率: 52.4%
- 待遇満足度: 2.3 / 士気: 2.3 / 風通し: 2.5 / 人事評価適正感: 2.7
- 年収レンジ: 800〜1200万円（推定）

### 核心固有性の選定と深掘り

```
■ 選定した核心固有性: 「インベストメント・チェーンの一員として持続可能な社会の形成に貢献する」

【源泉】
明治安田生命グループは「地域に根差した生命保険の相互扶助」という170年来の理念を持つ。
その子会社として、資産運用も「単なるリターン最大化」ではなく、
資金の流れを通じて社会全体の持続可能性に責任を持つという文化的背景から生まれた概念。
PRI（国連責任投資原則）に2016年3月署名、2024年次評価で4項目5つ星を獲得していること、
NZAM（ネット・ゼロ・アセット・マネージャーズ）に2023年1月加盟していることがこれを裏付ける。
※PRI署名自体の「日本最古参」はニッセイAM（2006年）。MYAMは先駆性ではなく評価内容（4項目5つ星）で語ること。

【競合差異】
野村AM・三井住友DSAMは「お客さまの資産形成支援」が主軸。
MYAMは「自社ファンドの保有者のため」だけでなく
「インベストメントチェーン全体（資本市場→企業→社会）の機能向上」を
公式ミッションとして掲げている点が異なる。

【面接での使い方】
「なぜこの企業か」の回答の"根拠・結び"として活用。
① なぜ資産運用か（起点：Fiduciary Duty・受益者責任）
② なぜ野村AMやニッセイAMではなくMYAMか（根拠：チェーン全体への責任意識、ALM視点）
③ なぜこのコースか（締め：エピソードとの接続）

【接続エピソード】
- `22` バスケ部守備の精神 → 「個人のリターンより全体のリスク管理」という哲学がチェーン責任と同じ構造
- `18` 卒論POSデータ因果推定 → 「一部のデータの偏りが全体推定を歪める」問題意識 ＝ 資本配分の歪みが社会全体に伝播するという世界観と接続
- `01` 全体最適志向 → 「個人最適ではなく最大多数の最大幸福」という価値観と直結

【注意点】
「インベストメントチェーン」は金融庁も使う一般用語だが、企業理念に据えて運用会社の
ミッションとして語るのはMYAMの文脈固有。
「業界用語を知っている」ではなく「なぜMYAMが使うのか」という背景（生保ALM・社会貢献軸）
まで語らないと暗記感が出る。
```

### F. 「総合職横断経験」志望動機への材料

#### 使える事実
1. **運用企画部が4機能を一手に担う**（経営戦略＋アロケーション＋クオンツAI＋責任投資）→ 「業務横断で顧客ニーズ→商品企画→運用判断まで」を一部門で経験できる実態がある
2. **ローテーション制度の実在**：口コミ・就活サイト複数で「様々な部署を経験できる」と確認。外部証券会社への訪問（営業視点の習得）も含む
3. **少数精鋭（219名）** = 1人の担当業務範囲が必然的に広い。企画・営業・運用の境界が薄い
4. **中谷社長（理工系・クオンツ出身）が運用・ESG・グローバル横断を一人で担当**：トップ自身が「横断型人材」のロールモデル
5. **Man Group提携で新商品開発・営業・運用連携が加速**：「顧客ニーズ（プライベートデット）→商品企画（外部委託設計）→運用（Man Groupと協業）」の回転が現実に動いている

#### 使える経営陣キャリア
- 中谷社長：損保（住友海上）→MYAM（クオンツ・ESG・グローバル株式→社長）。典型的な「他業界出身→AM横断キャリア」

#### 地雷パターン（避けるべき表現）
- ❌「他部門も経験したいので総合職を選びました」→ 主体性がなく見える。「横断経験が運用の質を高める理由」まで言わないと弱い
- ❌「御社はローテーションがあるから」→ 他AMでもあり得る一般論。「MYAMのどの部門を経験することで何が得られるか」を具体化すること
- ❌「営業→商品企画→運用の回転」を抽象的に言うだけ → 「Man Group提携でプライベートデット新商品の設計→機関投資家向け法人営業→運用での委託先モニタリング」という具体的な回転軸まで落とすこと

### G. 面接で使える非運用部門ファクトTOP5

1. **運用企画部は「経営戦略＋アロケーション＋クオンツAI＋責任投資」を4機能統合**した部門であり、AIモデルを内製開発中。「AI×運用融合」を志望する自分にとって直接の接点がある部門（運用職→運用企画部という移動ルートが志望と一致）

2. **Man Group提携（2025年2月）でプライベートデット商品の新設が進行中**。「商品企画部門がどのようなニーズでMan Groupの知見を商品化するか」というオペレーションが現在進行形 → 商品企画・法人営業・運用の連携が今まさに動いている案件として使える

3. **グループに証券会社がなく外部証券・銀行窓販チャネルに依存**。リテール営業職は「外部販売会社へのサポート」という業務で、純粋な顧客接点（投資家の声）を最前線で集める役割 → 「営業で集めた顧客ニーズを商品企画に還流させる」という回転の出発点として面白い

4. **クオンツAIモデルを「独自開発」と明言**し、社外コンテスト参加で外部の最新知見を取り込み中。少数精鋭（クオンツPM6名・平均経験13年）のチームが内製モデルを運用。「AIの設計者として参画したい」という志望と直接対応できる部門が社内に存在する

5. **中谷社長（東大理工系・クオンツ・ESG横断）が「AI×運用融合型人材」を評価できるトップ**。採用サイドのインセンティブとして「技術的素養を持ちながら運用哲学も理解できる人材」への期待値が高い可能性がある（直接言及はないが社長のバックグラウンドから推測）
