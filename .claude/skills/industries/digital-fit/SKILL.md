# digital-fit

## 前提
digital-commonの観点を前提に、金融機関のデジタル職固有の評価基準を追加する。

## 金融デジタル系で評価される特性

### 業界固有の評価軸
1. **金融ドメイン×デジタルの掛け算**
   - 金融の仕組みを理解した上でデジタルを使える
   - 「技術者」ではなく「金融×技術のプロ」
2. **内製化・変革への意志**
   - システムを「外注する人」ではなく「自分で作る人」になる意志
   - ベンダー依存からの脱却への共感
3. **非エンジニアとの協働**
   - ビジネス側・現場側に技術を分かりやすく説明できるか
   - 「翻訳者」としての役割への適性
4. **規制制約下のイノベーション**
   - 金融規制・個人情報保護と技術革新のバランス感覚

### ES・面接チェックポイント

**志望動機で必ず確認すること**
- [ ] 「なぜSIer/コンサルではなく金融機関IT職か」が語れるか
- [ ] 内製化・DXへの具体的なビジョンがあるか
- [ ] 金融ドメイン知識への関心が示されているか
- [ ] 技術だけでなく「誰の何を解決するか」が明確か

**NGパターン**
- 「プログラミングが好きだから」（目的ではなく手段として位置づけること）
- 「最新技術を使いたい」（金融ITはレガシーが多く、技術先行は現実離れ）
- 「フィンテックに興味がある」（どの機能・どの課題かが不明）

## 競合比較の視点
- IT戦略子会社の有無と役割（本体IT vs IT子会社）
- 内製化度合い・スタンスの差（各社中計での明示度）
- AI活用の実装段階（PoC vs 本番定着）
