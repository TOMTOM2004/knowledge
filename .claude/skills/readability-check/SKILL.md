# readability-check

## 目的
ESを「文系の若手面接官が初見で読んでわかるか」という基準で評価する手順。

## 使う場面
- ES草稿の可読性チェックをしたいとき
- 「わかりやすくして」と言われたとき

## チェックリスト

### 必須チェック（1つでも該当 → 要修正）
- [ ] 冒頭3行以内に「何をした人か」の結論がない
- [ ] 1文が70字を超えている文が2つ以上ある
- [ ] IT・金融・業界専門用語を説明なしに使っている
- [ ] 主語が不明確な文が2つ以上ある

### 推奨チェック（2つ以上該当 → 推奨修正）
- [ ] 「様々な」「多くの」などの曖昧な数量表現が3つ以上
- [ ] 「積極的に」「主体的に」などの行動形容詞が説明なしに使われている
- [ ] 「〜を通じて」「〜により」が1設問内で3つ以上
- [ ] カタカナ横文字が5つ以上（ポートフォリオ・イシュー・コミット等）

## 技術語変換辞書
| 元の表現 | 推奨変換 |
|---------|---------|
| アジャイル開発 | 少人数チームで短期間に繰り返し改善する開発手法 |
| KPI達成 | 数値目標の達成 |
| PDCAを回す | 計画・実行・確認・改善のサイクルを繰り返す |
| バリューチェーン | 価値を生み出す一連の業務プロセス |
| AUM | 運用資産総額 |
| フロントオフィス | 顧客と直接関わる営業・運用部門 |
| スキーム | 仕組み・枠組み |

## 出力形式
- チェックリスト評価結果
- 要修正箇所（行番号・該当文・改善案）
- 総合所見（1-2行）
