# 読者別の表現スタイル

ES・面接回答の読者（面接官）タイプに応じた文体・語彙の調整ガイド。

---

## 人事担当者向け（hr-readable-style）

**対象**: 採用経験3年以上、毎年数百枚のESを読むプロ

**人事が「よくある」と感じるパターンと差別化**

| よくあるパターン | 差別化の方向 |
|--------------|-----------|
| 「チームワークを学んだ」 | 具体的なチーム内での役割と変化を示す |
| 「主体的に取り組んだ」 | 誰かに言われずに始めた具体的な行動を書く |
| 「貴社でグローバルに活躍したい」 | どの業務でどの地域かを具体化する |
| 「課題を解決した」 | 課題の発見方法・解決アプローチの独自性を示す |

**人事が評価するポイント**
1. 一貫性: ガクチカ→自己PR→志望動機が同じ人物像
2. 具体性: 数字・名前・固有の場面
3. 成長性: 入社後どう成長しそうか
4. 本気度: この会社・職種を本当に調べているか

**チェックリスト**
- [ ] 「企業名を変えれば他社でも使える文」がないか
- [ ] ガクチカが「活動の説明」で終わっていないか
- [ ] 全設問を通じて同じ強み・価値観で一貫しているか
- [ ] 数字・固有名詞・具体的な場面描写があるか

## 文系若手面接官向け（junior-humanities-readable-style）

**対象**: 社会人1〜3年目の文系出身人事、金融・IT専門知識なし

**禁止表現と言い換え**

| 元の表現 | 言い換え |
|---------|---------|
| KPI達成 | 設定した数値目標の達成 |
| PDCAを回す | 計画・実行・振り返りのサイクルを繰り返す |
| アジャイル | 小さく試しながら改善を繰り返す手法 |
| AUM | 運用している資産の総額 |
| フィデューシャリー・デューティー | お客様の最善利益を最優先する義務 |
| バリュエーション | 企業の価値を数値で評価すること |
| ポートフォリオ | 複数の投資対象を組み合わせた運用全体 |
| DX | デジタル技術を活用した業務・価値提供の変革 |
| 内製化 | 外部に委託していた作業を自社で行うこと |

**文章スタイル**
- 1文は50字以内を目標
- 結論を最初の1〜2文に
- 接続詞の多用を避ける（「また」「さらに」「そして」の連続はNG）

**チェックリスト**
- [ ] 金融・IT専門用語が説明なしに使われていないか
- [ ] 1文が60字を超えていないか
- [ ] 冒頭3行で「この人が何をした人か」が伝わるか
- [ ] 「〜を通じて」が3回以上連続していないか

## 現場実務者向け（practitioner-readable-style）

**対象**: 志望職種の業務を10年以上経験している現場のプロ（2次〜最終面接）

**現場が求める表現**

| 表面的な表現 | 現場が求める表現 |
|------------|--------------|
| 「運用に携わりたい」 | 「アクティブ運用のFMとして○○セクターを担当したい」 |
| 「DXを推進したい」 | 「業務要件定義から実装まで自社で担う内製開発チームに携わりたい」 |
| 「法人営業で活躍したい」 | 「中堅企業の事業承継・M&Aニーズを起点に財務ソリューションを提案したい」 |

**現場が評価するポイント**
1. 業務理解の具体度: 日次業務・アウトプット・KPIを理解しているか
2. 論理的思考の証拠: 「なぜそう考えたか」の構造が見えるか
3. 課題設定力: 自分で課題を設定した経験があるか
4. 専門用語の正確な使用: 使うなら正確に（間違えると逆効果）

**専門用語の使い方原則**
- 正確に理解している用語のみ使う
- 曖昧な場合は使わずに言い換える
- 「使えることを見せようとした感」は逆効果

**チェックリスト**
- [ ] 業務内容を具体的に理解した上で書いているか
- [ ] 専門用語を正確に使っているか
- [ ] 判断・行動の根拠が論理的に示されているか
- [ ] 「教えてもらえれば何でもできます」ではなく「この点は既に持っている」が示せているか
