# 設問タイプ別の構造・答え方パターン

ES設問の種類ごとの構造テンプレート・配分・NG表現。

---

## 志望動機系（motivation）

### 3階層の「なぜ」

**Layer 1: なぜこの業界か**
```
[原体験・きっかけ] → [業界への関心の芽生え] → [自分のキャリアとの接続]
```

**Layer 2: なぜこの会社か**
```
[業界への共感（Layer1）] → [この会社の固有特徴] → [自分のビジョンとの接点]
```
※ research_briefの軸7（競合比較）が根拠

**Layer 3: なぜこの職種か**
```
[職種の業務理解] → [自分の強みとの対応] → [この職種で実現したいこと]
```

### 設問別の配分

| 設問 | 配分 |
|-----|------|
| 「志望動機を教えてください（400字）」 | L1(1/4)→L2(2/4)→L3(1/4) |
| 「なぜ当社を志望しますか（200字）」 | L2中心。L1・L3は1文ずつ |
| 「なぜこの職種を志望しますか」 | L3中心。職種理解→強みの対応→ビジョン |

### よくある弱点と対処

| 弱点パターン | 対処 |
|-----------|------|
| 競合でも言える内容 | research_brief軸7-3で固有根拠を補強 |
| 業界への関心はあるが会社への関心が薄い | 競合比較を追加 |
| 感情的すぎて根拠がない | 数字・具体的な事実で補強 |
| 職種理解が浅い | role研究で深掘り |

## ガクチカ系（gakuchika）

### 基本構造（STAR+why）

```
S（Situation）: いつ・どこで・どんな状況だったか（1〜2文）
T（Task）: 何が課題・目標だったか（1文）
A（Action）: 自分が具体的に何をしたか（2〜3文）★核心
R（Result）: どんな結果・成果を出したか（数字があれば必ず）
W（Why/Learning）: なぜその行動を取ったか・何を学んだか（1〜2文）
```

### 文字数別の配分

| 字数 | S | T | A | R | W |
|-----|---|---|---|---|---|
| 200字 | 省略気味 | 1文 | 2文 | 1文 | 1文 |
| 300字 | 1〜2文 | 1文 | 3文 | 1文 | 1〜2文 |
| 400字 | 2文 | 1〜2文 | 3〜4文 | 2文 | 2文 |

### よくある弱点

1. **活動の説明が中心でA（行動）が薄い** → 「自分がどう変えたか・動かしたか」を追加
2. **結果に数字がない** → 人数・件数・達成率・前後比較で定量化
3. **「頑張りました」が根拠** → 「なぜその方法を選んだか」の論拠を追加
4. **学びが抽象的** → 入社後のどの場面で活かすか具体的に接続

### 金融業界向けガクチカ評価基準
- 長期継続性（1年以上の取り組み）
- 定量的思考（数字で目標設定・評価）
- 責任ある役割（リーダー・幹事等）
- 困難への対処

## 強み・弱み・自己PR系（strengths-weaknesses）

### 強み設問の構造
```
[強みの定義（1文）]
→ [具体的な場面・エピソード（2〜3文）]
→ [結果（数字・評価）]
→ [職種・業務でどう活かすか（1〜2文）]
```

**強みのネーミング改善例**

| 曖昧な表現 | 具体的な表現 |
|-----------|-----------|
| リーダーシップ | 目標から逆算してチームを動かす力 |
| 分析力 | 数値データと定性情報を統合して判断する力 |
| コミュニケーション力 | 相手のニーズを引き出しながら合意を形成する力 |
| 粘り強さ | 成果が出るまで仮説と検証を繰り返す継続力 |

### 弱み設問の構造
```
[弱みの定義（1文）- 正直に、でも致命的でない弱みを選ぶ]
→ [弱みが顕在化した具体的な場面（1〜2文）]
→ [対処・改善策（2文）]
→ [現在の状況（改善中・改善済み）]
```

**弱みの選び方**
- 避ける: 志望職種の本質的能力に直結するもの
- 選ぶ: 改善努力が示せる、致命的でない弱み

**NGパターン**: 「弱みはありません」「頑張りすぎること」、弱みだけ言って対処法を述べない

### 自己PR設問との違い
- 強み設問: 1つの強みを深掘り
- 自己PR設問: 自分全体を売り込む（強み+エピソード+志望理由の入口）

## キャリアビジョン・将来像系（future-vision）

### 基本構造
```
[短期（1〜3年目）: 何を身につけるか]
→ [中期（3〜7年目）: どんな専門家になるか]
→ [長期（10年後〜）: 何を実現したいか]
→ [なぜそのビジョンをこの会社で実現したいか]
```

### 設問別の答え方
- 「入社後何をしたいですか」→ 1〜3年目の具体的な学習目標と業務イメージ中心
- 「10年後のビジョンを教えてください」→ 長期像中心。短期との接続も示す
- 「当社でのキャリアをどう描いていますか」→ 企業のキャリアパス（research_brief軸5）に沿ったロードマップ

### ビジョンの整合確認チェックリスト
- [ ] 企業のキャリアパス（research_brief軸5）と矛盾していないか
- [ ] 志望コースで実現可能なビジョンか
- [ ] 「なぜここでないとこのビジョンが実現できないか」の根拠があるか
- [ ] 1年目から10年後まで連続性のあるストーリーか

**NGパターン**: 「管理職になりたい」「グローバルに活躍したい」「社会に貢献できる人になりたい」、10年後の話だけして1年目・3年目の話がない

### 業界別のビジョン設計のコツ

| 業界 | ビジョン設計のポイント |
|-----|-------------------|
| AM | 特定アセットクラスのプロ→FM or チーフアナリスト |
| 銀行法人 | リレーションシップマネージャー→特定業種の専門家 |
| 証券リサーチ | 担当セクターのアナリスト→FM等 |
| DX・IT | 業務・技術の専門家→変革リーダー |
