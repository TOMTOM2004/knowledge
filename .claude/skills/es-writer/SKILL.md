# ES Writer

ユーザーから企業名・設問情報を受け取り、ナレッジベースを参照してESを生成し、
プレビュー提示 → 承認後に `移行後ES/<企業名>.md` に上書き保存 → 自動レビューまで一貫実行する。

## Step 概要

| Step | 内容 | 詳細 |
|------|------|------|
| 0 | 入力確認（企業名・ファイル特定） | `references/execution-protocol.md` |
| 1 | 移行後ESファイル読み込み・設問抽出 | 同上 |
| 2 | 業種判定・敬称決定 | 同上 |
| 3 | ナレッジ収集（並列: best_answers, INDEX, research_brief, 設問別部品） | 同上 |
| 4 | 設問種別マッピング → 回答生成 → G1-G11自己チェック | 同上 |
| 5 | 文字数チェック（95-100%目標） | 同上 |
| 6 | プレビュー提示（保存しない） | 同上 |
| 7 | ユーザー承認後に保存 | 同上 |
| 8 | mainブランチへコミット | 同上 |
| 9 | 自動レビュー（機械チェック → 設問適合 → 可読性 → 企業適合 → 重複） | 同上 |

## Core Rules

1. **research_brief.md 軸7-4（志望動機の核）** を志望動機の中心に置く。材料の羅列禁止
2. **企業固有情報不足は `【要確認】` タグ**で明示（推測で埋めない）
3. **抽象フレーム語禁止**:「構造化する」→「整理する」、「再現可能な仕組み」→「誰でも使える形」等
4. **文字数は95-100%**。未満なら補完、超過なら圧縮
5. **趣味特技は人柄優先**。無理なスキル連結禁止
6. **研究設問は完了/進行中を判定**してから時制を決める

## 注意事項

- `data/raw/` には絶対に触れない
- `ES/企業別/提出済/` は読み取り専用
- 文字数は `len()` で統一（全角・半角・記号すべてカウント）
