# digital-common

## 目的
デジタル・IT系職種に共通する評価観点・語彙・思考様式を提供する。
金融×デジタル（フィンテック、金融DX）の場合はfinance-commonと組み合わせて使う。

## デジタル系職種で評価される共通能力

### 思考特性
- **構造化思考**: 曖昧な課題を分解・整理して扱う
- **仮説検証思考**: 試行錯誤を前提にした学習サイクル
- **ユーザー思考**: 技術より「誰の何を解決するか」から出発する
- **変化適応力**: 技術・環境の変化を前向きに捉える

### 共通語彙（ESで使用可能・文系読者向け言い換え付き）
| 技術語 | 文系向け言い換え |
|-------|--------------|
| アジャイル開発 | 短いサイクルで改善を繰り返す開発手法 |
| データドリブン | データに基づいて意思決定する姿勢 |
| 内製化 | 外部委託していた開発・運用を自社で行うこと |
| DX推進 | デジタル技術で業務や価値提供を変革すること |
| PoC | 新技術や手法の小規模な実証実験 |
| ユーザーストーリー | 「誰が・何を・なぜ」を起点にした要件整理 |

### デジタル系のNG表現
- 「ITが好きだから」（業務理解の不足を示す）
- 「AIで何でも解決できる」（現実のシステム制約への無理解）
- 「プログラミングができます」（文系就活では技術自慢より課題解決視点）
- 「将来性があるから」（どの領域・どの機能かが不明）

## デジタル系共通の面接深掘り論点
1. **なぜSIerでなくユーザー企業ITか**: 内製化・ビジネス理解の重要性
2. **なぜコンサルでなく事業会社ITか**: 実装・運用まで関わる意志
3. **変化への適応**: 技術スタックが変わっても価値を出せるか
4. **文理融合**: 業務ドメイン知識とデジタルの掛け算をどう使うか

## 金融×デジタルの場合の追加観点
- 規制制約下でのイノベーション（個人情報・説明可能性）
- レガシーシステムとモダン技術の共存
- フィデューシャリー・デューティーとAI活用の整合
