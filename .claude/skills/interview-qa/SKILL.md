# interview-qa スキル

## 責務

**このスキルがやること**: 重要度別の回答骨格生成・深掘り展開・詰められ対策・逆質問リスト作成  
**このスキルがやらないこと**: 体験記の収集・質問の重要度分類（→ interview-research が担当）

**前提**: `interview-research` を先に実行し、`interview_research_<ステップ>_*.md` が存在すること

**参照ファイル（必要時に読み込む）:**
- `references/answer-patterns.md` — 面接回答パターン（話し言葉版）。ガクチカ/志望動機/強み弱み/深掘り/反論対処（Step 2で回答骨格生成時に読み込む）
- `references/stage-modes.md` — 選考段階別の評価基準・チェックリスト（Step 1で選考ステップ特定後に読み込む）
- `references/role-fit-criteria.md` — 職種別の評価観点・頻出質問（Step 2でコース特定後に該当セクションを読み込む）

---

## Step 0: 入力確認

以下を確認する。不明な場合はユーザーに質問する。

| 項目 | 例 |
|-----|-----|
| 企業名 | 〇〇証券 |
| コース・職種 | 総合職 / 営業コース |
| 選考ステップ | 1次面接 / 2次面接 / 最終面接 |

---

## Step 1: 入力ファイルの読み込み（並列）

以下を並列で読み込む。

1. `company-info/<企業名>/interview_research_<ステップ>_*.md` — 最新ファイル
   - ファイルが存在しない場合: `【警告】interview_research ファイルが見つかりません。interview-research スキルを先に実行してください。続行しますか？` と表示し、確認を求める
2. `company-info/<企業名>/research_brief.md` — 軸3・軸7-A・軸8・軸9 を抽出
3. `移行後ES/<企業名>.md` — 提出済み回答を確認（存在しない場合はスキップ）
4. `ES/md/01_自己分析_コア価値観将来像行動原理.md` — コア価値観・行動原理を抽出
5. `ES/md/02_自己分析_強み弱み伸びしろ.md` — 強み・弱みを抽出

---

## Step 2: 重要度別の回答骨格生成

### [高] 質問（フル生成）

重要度「高」の各質問に対して以下をすべて生成する:

```
【高】<質問文>

■ 回答骨格
- 結論: <一言で答え>
- エピソード: <使うエピソードのタイトル>（ES/md/INDEX.md の番号）
- 具体内容: <数字・固有名詞・アクション>
- 締め: <この企業・コースに接続する一言>

■ 自己接続チェック
- この企業の何と繋がるか: <research_brief.md 軸7-A または軸3 の記述>
- エピソードがES提出済みの場合: <面接での言い換えポイント>

■ 深掘り展開（2段）
1段目: <想定深掘り質問> → <回答の方向性>
2段目: <さらなる深掘り質問> → <回答の方向性>

■ 反論耐性チェック
- 想定される反論: <面接官が詰めてくるポイント>
- 対策: <切り返し方の方向性>
```

### [中] 質問（標準生成）

重要度「中」の各質問に対して以下を生成する:

```
【中】<質問文>

■ 回答骨格
- 結論: <一言で答え>
- エピソード: <使うエピソードのタイトル>
- 具体内容: <数字・固有名詞・アクション>
- 締め: <この企業・コースに接続する一言>

■ 深掘り展開（2段）
1段目: <想定深掘り質問> → <回答の方向性>
2段目: <さらなる深掘り質問> → <回答の方向性>
```

### [低] 質問（簡略生成）

重要度「低」の各質問に対して以下のみ生成する（深掘りは省略）:

```
【低】<質問文>
- 結論: <一言で答え>
- 使うエピソード: <タイトル>
- 要点: <2〜3行>
```

---

### 志望動機系回答の特別ルール

志望動機・「なぜこの企業か」・「なぜこのコースか」に関する質問には、
`research_brief.md` の軸7-A（核心固有性）を必ず組み込む。

```
志望動機の構成:
1. 核心固有性（軸7-A）を起点にする
2. 自分のエピソード・価値観との接続
3. このコースでやりたいこと（軸2・軸5を参照）
```

軸7-A が research_brief.md に存在しない場合:
`【固有性未抽出】company-researcher スキルの軸7-A を実行してから再実行することを推奨します。`
と表示し、続行確認をする。

## Step 4: 逆質問リスト（4カテゴリ×2問）

以下の4カテゴリでそれぞれ2問の逆質問を生成する。
軸7-A（核心固有性）や軸1-D（最重要経営課題）からフックを作ることを優先する。

### カテゴリ1: 事業・戦略への関心
```
Q: <質問>
フック: <research_brief.md の軸1または軸7-A の記述>
```

### カテゴリ2: コース・業務の具体像
```
Q: <質問>
フック: <research_brief.md の軸2または軸4 の記述>
```

### カテゴリ3: キャリア・成長環境
```
Q: <質問>
フック: <research_brief.md の軸5 の記述>
```

### カテゴリ4: 面接官個人への質問
```
Q: <面接官の仕事観・価値観に触れる質問>
フック: <面接官の年次・スタイルから設計>
```

## Step 6: 保存

承認後、以下のパスに保存する:

```
company-info/<企業名>/interview_qa_<ステップ>_<YYYYMMDD>.md
```

例: `company-info/〇〇証券/interview_qa_1次面接_20260403.md`

保存後:
```
保存完了: company-info/<企業名>/interview_qa_<ステップ>_<YYYYMMDD>.md

次のステップ:
本番練習 → 「<企業名>の<N>次面接で skeptical-interviewer を回して」（別セッション推奨）
```

## 品質ガードレール

- 重要度「高」・「中」の質問には深掘り2段展開を必須とする
- 重要度「低」の質問は深掘り省略（骨格のみ）
- 志望動機系回答には軸7-A の核心固有性を必ず組み込む
- 軸7-A が未抽出の場合は `【固有性未抽出】` タグをつけ company-researcher 再実行を推奨する
- 回答骨格は箇条書きのみ（文章化しない）—— 暗記を防ぐため
- interview_research ファイルなしで実行する場合は警告を出してから続行確認する
