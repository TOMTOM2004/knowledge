# Interview Prep

企業名・コース・選考ステップを受け取り、面接準備シートを生成する。
`research_brief.md` を起点とし、Web調査で選考情報・頻出質問を補完する。

**7つの調査軸**: 実態確認 / 根拠確認 / 抽象語の具体化 / 実装方式 / 現場業務 / 突っ込まれポイント / 自己接続

## Step 概要

| Step | 内容 | 詳細 |
|------|------|------|
| 0 | 入力確認（企業名・コース・ステップ） | `references/execution-protocol.md` |
| 1 | 社内ナレッジ読み込み（research_brief, 自己分析） | 同上 |
| 2 | Web調査（5グループ並列: A選考実態 B数値裏取り C抽象語具体化 D業界比較 E中計IR差分） | 同上 |
| 2.5 | 調査結果の7軸構造整理 | 同上 |
| 3 | ステップ別・想定質問リスト生成（5カテゴリ） | 同上 |
| 4 | 回答骨格生成（深掘り問答・反論耐性チェック付き） | 同上 |
| 5 | 詰められポイントと対策（10観点） | 同上 |
| 6 | プレビュー提示 | 同上 |
| 7 | ユーザー確認後に保存 | 同上 |

## Core Rules

1. **「なぜこの企業か」** の回答骨格は軸7（競合比較）の内容を必ず盛り込む
2. **「なぜこの業界か」** は業界前段（⑥）を必ず踏まえる
3. **回答骨格は箇条書き**（文章化しない）
4. **抽象語禁止**: 「貢献」「活躍」「成長」「DX推進」等は具体行動レベルに言い換え
5. **深掘り問答は最低2段階**展開
6. **未確認情報は `【要確認】` タグ**。根拠不明の数値には `【根拠不明】`
