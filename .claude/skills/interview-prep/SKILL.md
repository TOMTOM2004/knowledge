# Interview Prep スキル

企業名・コース・選考ステップ（何次面接か）を受け取り、面接準備シートを生成する。
company-researcher の `research_brief.md` を起点とし、Web調査で選考情報・頻出質問を補完する。
生成したシートは `company-info/<企業名>/interview_prep_<ステップ>_<日付>.md` に保存する。

**このスキルが最初から対象とする7つの調査軸:**
1. 実態確認（公式 vs. 現場の実運用）
2. 根拠確認（数値・言説の出典と算定ロジック）
3. 抽象語の具体化（DX・AI・CX等を業務レベルへ分解）
4. 実装方式の確認（内製/外部/API連携/セキュリティ審査）
5. 現場業務への落とし込み（若手が触る業務・年次別役割）
6. 面接での突っ込まれポイントと反論耐性
7. 自己接続（自分の経験・強み・価値観との対応）

---

## Step 0: 入力確認

ユーザーの依頼から以下を特定する。

- **企業名**: 依頼文から抽出
- **志望コース・部門**: 依頼文から抽出
- **選考ステップ**: 「1次面接」「2次面接」「最終面接」など

いずれかが不明な場合は調査を開始する前にユーザーに確認する。

```
「以下を確認させてください。
・志望するコース・部門（例：リサーチコース、運用部門）
・選考ステップ（例：1次面接・2次面接・最終面接）」
```

## Step 2: Web調査（並列・7軸対応）

以下のクエリグループを並列で実行する。各グループの目的を意識して情報を整理すること。

### グループA: 選考実態・求める人材像の確認（公式情報 + 体験談）

| 観点 | 推奨クエリ |
|------|-----------|
| 選考体験記・頻出質問 | `<企業名> <コース名> 面接 体験記 質問 2024 2025` |
| 選考フロー・形式 | `<企業名> 新卒 選考フロー 面接 何回` |
| 面接形式の実態 | `<企業名> <ステップ> 個人面接 グループ面接 人数` |
| 面接官の実態 | `<企業名> <ステップ> 面接官 人事 現場 年次` |
| 求める人材像（公式） | `<企業名> 求める人材 人材像 採用方針 新卒` |
| 求める人材像（体験談） | `<企業名> 面接 評価 重視 どんな人 雰囲気 合否` |

> **実態確認の観点**: 公式説明と体験談が一致しているか、例外ケースはないか、当日の変動要素（急な形式変更など）はないかを確認する。矛盾があれば両論を並記し【要確認】タグをつける。

> **求める人材像の観点**: 公式採用ページ・社員インタビュー・体験談の「どんな人が受かっていたか」を突き合わせて実態を把握する。`research_brief.md` の軸3が存在する場合はその内容と照合し、差異があれば両論を並記する。抽象的な表現（「主体性」「チャレンジ精神」等）が出たら、**企業が具体的にどんな行動・エピソードを評価しているか**まで落とし込む。

### グループB: 数値・根拠の裏取り

| 観点 | 推奨クエリ |
|------|-----------|
| IR・中期経営計画 | `<企業名> 中期経営計画 IR 数値 目標 2024 2025` |
| 実績・成果根拠 | `<企業名> 業績 事例 成果 実績 根拠` |
| 企業の主張の裏取り | `<企業名> <主要施策名> 効果 根拠 実績` |

> **根拠確認の観点**: 数値の出典・定義・算定根拠を確認する。「○○億円のシナジー」「顧客満足度No.1」などの表現が出たら、何に基づくのかをメモする。根拠不明の場合は【根拠不明】タグをつける。

### グループC: 抽象語の具体化・現場業務

| 観点 | 推奨クエリ |
|------|-----------|
| コース別の具体業務 | `<企業名> <コース名> 仕事内容 業務 具体的` |
| 若手の実務 | `<企業名> 若手 入社 仕事 1年目 3年目` |
| DX・AI施策の実態 | `<企業名> DX AI デジタル 具体的 何をしているか` |
| AI実装形態 | `<企業名> AI 生成AI 外部サービス 内製 API 活用` |

> **抽象語の具体化の観点**: 「DX推進」「AI活用」「顧客体験向上」などの抽象表現が出たら、「誰が・何を・どうやって・何が変わるのか」まで分解して記録する。わからなければ【具体化要】タグ。

> **実装方式の観点**: AIやデジタルツールについては、内製/外部SaaS/API連携/生成AI（OpenAI等）の別、セキュリティ審査の有無、現場エンジニアの利用自由度まで確認する。

### グループD: 業界志望の前段・競合比較

| 観点 | 推奨クエリ |
|------|-----------|
| 業界特性・社会的役割 | `<業界名> 役割 社会的意義 特徴 他業界との違い` |
| 業界選択の差別化 | `<業界名> vs <隣接業界> 違い 特徴` |
| 競合との差分 | `<企業名> vs <競合他社> 強み 特徴 違い` |
| 最近の企業ニュース | `<企業名> ニュース 2025 新規事業 戦略` |

> **業界志望の観点**: 「なぜ銀行か」「なぜこの業界か」という前段の質問に答えられるよう、その業界でしか得られない価値・社会的役割・他業界との差分を整理する。

### グループE: 中期経営計画・IR差分分析（推進内容と達成状況の乖離）

| 観点 | 推奨クエリ |
|------|-----------|
| 中期経営計画の全体像 | `<企業名> 中期経営計画 目標 KPI 重点施策 site:<企業公式ドメイン>` |
| 今期の業績・進捗報告 | `<企業名> 決算 業績 中計 進捗 達成 2025` |
| 計画修正・下方修正 | `<企業名> 中計 修正 下方修正 計画変更 見直し 2024 2025` |
| 未達・課題の開示 | `<企業名> IR 課題 未達 遅れ リスク 懸念 2024 2025` |
| 撤退・縮小・方針転換 | `<企業名> 撤退 縮小 事業売却 方針転換 見直し` |

> **中計・IR差分分析の観点**: 以下の4点を必ず整理する。
> 1. **推進中の重点施策**: 中期経営計画で掲げている主要テーマ・KPI・投資領域
> 2. **現状の達成度**: 今期開示資料で確認できる進捗・達成率・修正後目標
> 3. **できていないこと（ギャップ）**: 計画比で遅れている・未達の領域、撤退・縮小した施策、開示されているリスク・課題
> 4. **差分の構造的理由**: なぜできていないのか（外部環境変化・内部実行力・市場想定の誤り等）
>
> 面接での活用方法: 「推進しているが苦戦している領域」は「自分が貢献できる余地」として志望動機・入社後にやりたいことに接続できる。逆質問の素材にもなる。

## Step 2.5: 調査結果の構造整理

Web調査・社内ナレッジの情報を以下の7軸フレームに整理してから、Step 3以降の生成に進む。

```
### 調査結果サマリー

#### ① 公式情報
- 制度・仕事内容・会社の主張・募集要項の要点

#### ② 求める人材像
- 公式採用ページ・社員インタビュー・IR等に記載された求める人物像
- 体験談ベースで「合格者に共通していた特徴・評価されていた行動」
- 抽象表現（主体性・チャレンジ精神等）を具体行動レベルに落とした解釈
- `research_brief.md` 軸3との照合結果（差異があれば両論並記）

#### ③ 実態（体験談ベース）
- 体験談から把握できた実際の選考形式・面接官・現場運用
- 公式情報との乖離・例外ケース

#### ③ 根拠確認
- 数値の出典と定義（【根拠不明】タグがある項目は列挙）
- 主要な言説の裏取り結果

#### ④ 抽象語の具体化
- DX・AI・CX等の抽象表現を業務レベルに分解した結果
- 実装方式（内製/外部/API連携/生成AI）の確認結果

#### ⑤ 現場業務
- コース別の主要業務
- 若手（入社1〜3年目）が実際に担う業務
- 年次ごとの役割変化

#### ⑥ 業界志望の前段
- この業界でしか得られない価値・社会的役割
- 隣接業界・競合との差分

#### ⑦ 中計・IR差分分析
- **推進中の重点施策**: 中計で掲げる主要テーマ・KPI・投資領域（出典URLまたはIR資料名を併記）
- **現状の達成度**: 今期開示資料で確認できる進捗・達成率・修正後目標
- **できていないこと（ギャップ）**: 遅れ・未達・撤退・縮小した領域と、開示されているリスク・課題
- **差分の構造的理由**: 外部環境変化・内部実行力・市場想定の誤り 等
- **面接活用メモ**: このギャップが志望動機・入社後にやりたいこと・逆質問にどう接続できるか

#### ⑧ 【要確認】リスト
- 確認できなかった情報・矛盾が残る点
```

## Step 3: 選考ステップ別・想定質問リストの生成

ステップに応じて質問の重み付けを変える。

### ステップ別フォーカス

| ステップ | 重点カテゴリ |
|---------|------------|
| **1次面接** | 自己紹介・自己PR・ガクチカ（人柄・基本スペックの確認） |
| **2次面接** | ガクチカ深掘り・失敗談・チームワーク・志望動機の確認 |
| **最終面接** | なぜこの企業か・キャリアビジョン・入社後にやりたいこと・逆質問 |

### 質問カテゴリ（全ステップ共通の5カテゴリ）

各カテゴリから、ステップに応じて3〜6問を選定する。

**カテゴリA: 自己紹介・自己PR**
- 1分で自己紹介してください
- あなたの強みを教えてください
- あなたを一言で表すと？

**カテゴリB: 志望動機系**
- なぜこの業界を志望するのか（業界前段を必ず含む）
- 同業他社ではなく、なぜ弊社を志望するのか
- なぜこのコース・部門を選んだのか

**カテゴリC: ガクチカ・エピソード系**
- 学生時代に最も力を入れたこと（ガクチカ）
- チームで困難を乗り越えた経験
- 失敗した経験と、そこから学んだこと

**カテゴリD: キャリア・将来像**
- 入社後にやりたいこと（3年・5年・10年）
- どのような人材になりたいか
- 弊社でのキャリアビジョンを聞かせてください

**カテゴリE: 深掘り・逆質問**
- （ガクチカ・エピソードへの深掘り想定質問）
- 弱みはなんですか、どう克服していますか
- 面接官への逆質問（3〜5個）

## Step 4: 回答骨格の生成（深掘り問答付き）

Step 3 で選定した各質問に対して、以下の形式で回答骨格を作成する。

```
### Q: <質問文>

**回答骨格:**
- 結論（1文）: ...
- 根拠・エピソード: <使うエピソード名>（例: 03_インターン_財務資料自動化）
- 具体的内容（箇条書き）:
  - ...
  - ...
- 締め・企業との接点: ...（Step 2.5の⑤現場業務・⑥業界前段を踏まえて具体化）

**自己接続チェック:**
- 自分の強み・価値観のどこと接続しているか: ...
- 使っている表現が抽象的すぎないか（危険ワード: 「貢献」「活躍」「成長」等）: ...

**深掘り問答（想定チェーン）:**
- 深掘りQ1: <この回答に対して面接官が次に問いそうな質問>
  - 返答骨格: ...
- 深掘りQ2: <さらに突っ込まれた場合>
  - 返答骨格: ...
- 危険ワード・抽象語が残っている場合の先回り補足: ...

**反論耐性チェック:**
- 「それは他社でも言えますよね？」への返答: ...
- 「それはあなたでなくてもできたのでは？」への返答: ...（該当する場合）
- その他の主な反証・例外への対応: ...
```

**エピソード選定の優先順位**
- research_brief.md の軸8で「◎ 強くマッチ」→「○ マッチ」の順で使用
- research_brief.md がない場合は自己分析ファイルから判断

## Step 5: 詰められポイントと対策（10観点）

想定される厳しい質問・突っ込みを事前に洗い出し、対策を示す。
以下の10観点を必ず確認する。

| # | チェック観点 | 詰められポイントの例 |
|---|------------|------------------|
| 1 | **エピソードの数値・根拠** | 「16時間とはどう測定したか」「100%とはどういう基準か」 |
| 2 | **志望動機の競合差別化** | 「それは〇〇社でもできますよね？」（軸7で補強） |
| 3 | **強み×企業の接点の薄さ** | 「その強みはうちで具体的にどう活きますか」 |
| 4 | **弱みの実在性** | 「その弱みは克服できましたか。今も出ますか」 |
| 5 | **キャリアの一貫性** | 「なぜ学生時代にDSをやっていたのに運用志望なのか」 |
| 6 | **ガクチカの規模・独自性** | 「それはあなたでなくてもできたのでは」 |
| 7 | **抽象語の危険** | 「意思決定の質向上とは具体的に何ですか」「CXとは何を指しますか」 |
| 8 | **業界志望の前段の弱さ** | 「なぜ銀行でないといけないのですか」「保険でなく証券では駄目なのですか」 |
| 9 | **実態との乖離** | 「そのコースで実際に担う業務は知っていますか」「入社後すぐにその業務はできません」 |
| 10 | **反論・例外への対応** | 「その施策がなかったら何をしたいですか」「それは広すぎる言い方では」 |

各詰められポイントに対して:
```
**リスク:** <突っ込まれる内容>
**対策:** <回答の方向性（箇条書き）>
**先回り補足:** <回答骨格のどこを補強すれば防げるか>
```

## 品質ガードレール

- 「なぜこの企業か」の回答骨格は必ず軸7（競合比較）の内容を盛り込む。競合でも言える回答はNGとして指摘する
- 「なぜこの業界か」の回答骨格は必ず業界前段（Step 2.5の⑥）を踏まえて書く。業界選択の前段が薄ければ【要補強】タグをつける
- 回答骨格は箇条書きで完結に留める。文章化はしない
- 抽象語（「貢献」「活躍」「成長」「DX推進」「AI活用」「CX向上」等）が回答骨格に残っている場合は必ず具体行動・成果指標レベルに言い換える
- 深掘り問答は最低2段階（初回質問 → 深掘りQ1 → 深掘りQ2）まで展開する
- 詰められポイントは楽観的にならず、面接官視点で厳しめに設定する
- 未確認情報は【要確認】タグをつけ、断定しない。根拠不明の数値には【根拠不明】タグ
- 逆質問は「調べればわかること」は避け、面接官の思考・判断・現場実態を引き出す質問を優先する
- 自己接続チェックが「表面的な接続」（経験と企業のキーワードが似ているだけ）にならないよう、価値観・行動原理レベルで一致しているかを確認する
