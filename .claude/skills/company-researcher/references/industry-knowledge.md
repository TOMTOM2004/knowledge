## 金融業界共通（finance-common）

### 金融業界で評価される共通能力

**思考特性**
- 数字への親和性: 感覚ではなくデータ・根拠で考える姿勢
- リスク感覚: リターンとリスクのバランスで判断する習慣
- 長期視点: 短期の数字より長期の価値創造を重視する視点
- 誠実さ・正確性: 顧客の資産・信頼を扱う職業倫理

**共通語彙（ESで使用可能）**

| 一般語 | 金融適合表現 |
|-------|-----------|
| お客様の役に立つ | 顧客の資産形成・財務目標の実現に貢献する |
| 情報収集が好き | 経済・市場情報のアンテナを張り、意思決定に活かす |
| 課題解決 | 顧客の財務課題に対するソリューション提供 |
| 分析が得意 | 定量データと定性情報を統合して判断する |

**金融業界NG表現**
- 「安定しているから」「お金が好きだから」「金融に興味があります」「社会に貢献できる」

**面接深掘り論点**
1. なぜ金融か（製造業・コンサル・商社との違い）
2. なぜこの機能か（運用/銀行/証券の機能差）
3. 規制環境への理解
4. デジタル化への認識

## デジタル・IT系共通（digital-common）

### デジタル系職種で評価される共通能力

**思考特性**
- 構造化思考: 曖昧な課題を分解・整理して扱う
- 仮説検証思考: 試行錯誤を前提にした学習サイクル
- ユーザー思考: 技術より「誰の何を解決するか」から出発する
- 変化適応力: 技術・環境の変化を前向きに捉える

**共通語彙（文系向け言い換え付き）**

| 技術語 | 文系向け言い換え |
|-------|--------------|
| アジャイル開発 | 短いサイクルで改善を繰り返す開発手法 |
| データドリブン | データに基づいて意思決定する姿勢 |
| 内製化 | 外部委託していた開発・運用を自社で行うこと |
| DX推進 | デジタル技術で業務や価値提供を変革すること |
| PoC | 新技術や手法の小規模な実証実験 |

**デジタル系NG表現**
- 「ITが好きだから」「AIで何でも解決できる」「プログラミングができます」「将来性があるから」

**面接深掘り論点**
1. なぜSIerでなくユーザー企業ITか
2. なぜコンサルでなく事業会社ITか
3. 変化への適応
4. 文理融合の掛け算

## アセットマネジメント業界

### AM業界固有の評価軸
1. **受託者責任（フィデューシャリー・デューティー）への理解**
2. **長期・複利思考**: 短期の変動より長期のリターン最大化
3. **情報の非対称性への対処**: 一般投資家が知りえない情報・分析を提供する使命感
4. **市場・経済への知的好奇心**: 相場・経済ニュースへの継続的関心

**コース別評価軸**

| コース | 追加評価軸 |
|--------|-----------|
| 運用（FM） | 判断力・決断力・ポートフォリオ思考 |
| リサーチ | 深掘り・数値モデル・企業分析 |
| 営業 | 課題把握・提案力・信頼構築 |
| IT・デジタル | 金融×IT の掛け算理解 |

**志望動機チェック**
- [ ] 「運用業とは何か（信託報酬モデル・受益者利益）」を理解しているか
- [ ] 銀行・証券との機能差が明確か
- [ ] 「長期×複利」の思想への共感が具体的か
- [ ] AUM規模・競合との差別化が1点以上あるか

**NGパターン**: 「投資に興味がある」「安定した仕事」「多くの人の資産を守りたい」

**競合比較の視点**: 独立系 vs 銀行系 vs 生保系 vs 外資系 / 運用哲学 / AUM規模・顧客層 / 特定の強み領域

## 銀行業界

### 銀行固有の評価軸
1. **長期リレーション構築力**: 「取引」ではなく「パートナーシップ」の視点
2. **課題発見・ソリューション提案力**: 「御用聞き」ではなく「課題解決者」
3. **泥臭さ・粘り強さ**: 法人営業は断られてから始まる仕事
4. **規制・コンプライアンスへの感度**

**コース別評価軸**

| コース | 追加評価軸 |
|--------|-----------|
| 法人営業 | 企業財務理解・M&A/財務コンサル志向 |
| リテール | 個人の生涯設計への関心 |
| マーケット | 市場分析・スピード判断・リスク管理 |
| 信託・資産管理 | 相続・遺言・不動産への専門性 |
| IT・デジタル | digital-common 参照 |

**志望動機チェック**
- [ ] 「間接金融としての銀行の機能」を理解しているか
- [ ] 証券・AM・保険との機能差が明確か
- [ ] 「なぜメガ・地銀・信託・信金か」が具体的か

**NGパターン**: 「銀行は社会インフラだから」「グローバルに活躍したい」「融資で企業の成長を支える」

## 証券業界

### 証券固有の評価軸
1. **市場・情報への高感度**: 「情報を仕入れて価値に変換する」
2. **スピード判断・実行力**: 「仮説→実行→修正」のサイクル
3. **直接金融への使命感**: 企業の資金調達を支える「マーケットの橋渡し役」
4. **数字・パフォーマンスへの執着**

**コース別評価軸**

| コース | 追加評価軸 |
|--------|-----------|
| リサーチアナリスト | 企業分析・モデリング・論理的文章 |
| セールス | 信頼構築・深い分析提供・長期フォロー |
| マーケット・トレーディング | 瞬時判断・リスク管理・数値センス |
| IB | M&A・資本市場の知識・高密度作業耐性 |

**志望動機チェック**
- [ ] 「直接金融vs間接金融」の機能差を理解しているか
- [ ] 「なぜこの証券会社か」の固有根拠があるか
- [ ] 変動の激しい環境への適応意志が示されているか

**競合比較**: 野村/大和/SMBC日興の差別化 / 外資系との違い / ネット証券との差別化

## デジタル・DX系職種（金融機関向け）

### 金融デジタル系固有の評価軸
1. **金融ドメイン×デジタルの掛け算**: 「金融×技術のプロ」
2. **内製化・変革への意志**: 「自分で作る人」になる意志
3. **非エンジニアとの協働**: 「翻訳者」としての役割
4. **規制制約下のイノベーション**

**志望動機チェック**
- [ ] 「なぜSIer/コンサルではなく金融機関IT職か」が語れるか
- [ ] 内製化・DXへの具体的ビジョンがあるか
- [ ] 「誰の何を解決するか」が明確か

**NGパターン**: 「プログラミングが好き」「最新技術を使いたい」「フィンテックに興味がある」

**競合比較**: IT戦略子会社の有無 / 内製化度合い / AI活用の実装段階
