# 業界・職種別リサーチレンズ

company-researcher が調査を深める際に参照する、業界・職種固有の追加調査観点。
必要な業界セクションだけを読めばよい。

---

## アセットマネジメント

### AM固有の追加調査観点（4群構造）

> 競合横断比較の詳細軸定義: `docs/am-comparison-framework.md`（28軸・3層モデル）
> 横断比較スキル: `am-competitor-researcher`

#### A群: 事業・顧客・商品

- **顧客構成**: 年金・機関投資家・リテール・グループ内の比率。外部顧客比率
- **販売チャネル構造**: 親会社販売網への依存度。独自で証券会社に卸すか
- **グループ内役割分担**: AM子会社 vs 本社運用部門 vs IT子会社の棲み分け
- **母体の法的形態**: 相互会社（契約者に責任）vs 株式会社（株主に責任）
- **パッシブ/アクティブ比率**: 数値ベースで確認
- **旗艦商品**: 収益の柱になっている商品・ファンド
- **Alternative positioning**: オルタナティブを成長領域として見ているか / 自社運用か外部マネージャー活用か
- **Asset growth quality**: AUM絶対額ではなく、どの顧客層で伸びているか / 純流入の有無
- **Fee resilience**: 低コスト競争への耐性 / 高付加価値領域の有無 / 収益の厚み

#### B群: 運用の中身

- **Alpha source**: 超過収益の源泉。何が株価にまだ織り込まれていないと考えるか
  - └ **Information edge**: 情報優位の源泉 / 調査への投資度合い / 定量・定性・データの接続
  - └ **Alternative data utilization**: 何の仮説にどう使うかが重要（言及の有無だけでは不十分）
- **Research architecture**: PM主導かアナリスト主導か / セクター制かテーマ制か / クオンツの位置づけ
- **Decision rights**: FM個人裁量か委員会型か / エンゲージメント担当の意見が投資判断に入るか
- **Portfolio construction**: 集中か分散か / Active Share / 利確・損切り規律
- **Time horizon**: 投資時間軸。エンゲージメントを掲げていても時間軸が短ければ本当の企業変革ではない
- **Risk management philosophy**: 下落局面への基本姿勢 / 損切り規律 / リスク管理を守りと捉えるか超過収益の一部と捉えるか

#### C群: スチュワードシップ

- **Stewardship integration**: エンゲージメントが投資仮説の一部か / 対話が運用判断を変えたか
- **議決権行使方針**: 各社公開義務あり。行使方針の違いはAMの「色」が最も出る
- **対話テーマ・実績**: どの企業に / 何を論点に / 組入比率がどう変わったか
- **Stewardship autonomy**: 自前でどこまで判断しているか
  - └ **Proxy advisor dependence**: ISS/Glass Lewis への依存度 / 議決権行使説明の具体性

#### D群: 持続可能性

- **Economics/incentives**: 手数料のどこで稼いでいるか / アクティブをやる経済合理性
- **Talent model**: アナリスト→PM昇格ルート / 内部育成か外部採用依存か
  - └ **Talent pipeline proxy**: 新卒採用の継続性 / 中途比率 / 育成か即戦力依存か（補助項目）
- **親会社シナジー/制約**: 安定受託基盤の反面、独立性の制約
- **海外ネットワーク**: 海外拠点 / 外部委託比率 / グローバルリサーチへのアクセス
- **Organizational health**: 口コミ / 勤続年数 / 若手定着 / 建前と実態のズレ（補助項目）

**クエリテンプレート**
```
[企業名] 運用哲学 アクティブ パッシブ 方針
[企業名] AUM 残高 推移 純流入 2024
[企業名] オルタナティブ プライベートアセット 戦略
[企業名] 信託報酬 料率 低コスト 収益構造
[企業名] スチュワードシップ報告書 議決権行使 2024
[企業名] フィデューシャリー 顧客本位 方針
[企業名] 競合 野村AM ニッセイAM 大和AM 比較
```

**差別化ポイントの発見方法**
1. 「取り組みの有無」ではなく「価値創出の仕組みの違い」を見る
2. 運用哲学の独自性 — 他社と何が違うか（B群で差がつく）
3. 収益構造の耐性 — 低コスト競争に巻き込まれない構造があるか（A-9）
4. スチュワードシップの本気度 — 自前判断か助言会社依存か（C-4）
5. AUM成長の質 — 純流入か市場上昇だけか（A-8）

## 銀行

### 銀行固有の追加調査観点

**事業構造**
- コア業務の比重: 融資業務 / 投資業務 / 手数料業務の比率
- 法人 vs リテールの比重
- 海外展開の規模・拠点
- グループシナジー: 証券・信託・保険との連携状況

**中期経営計画の重点施策**
- DX・システム刷新への投資額と方向性
- 収益改善策（低金利環境での収益多様化）
- 人材戦略（スペシャリスト化・外部採用）

**競合比較（メガ3行 + 信託）**
```
[企業名] 法人融資 強み 中小企業 大企業 比率
[企業名] 海外展開 拠点 ASEAN アメリカ 規模
[企業名] DX 内製化 システム 投資 2024
三菱UFJ vs みずほ vs 三井住友 強み 比較 新卒
[企業名] 信託 信託業務 強み 残高
```

**コース別の追加調査**

| コース | 追加で調べること |
|--------|--------------|
| 法人営業 | 主要取引先の業種・大企業vs中堅の比率 |
| リテール | 資産運用・投資信託販売への注力度 |
| マーケット | 市場部門の規模・運用残高 |
| IT・デジタル | ITシステムの内製化状況・DX子会社との関係 |
| 信託 | 年金・不動産・相続信託の残高 |

**差別化ポイントの発見方法**
1. メガ3行の中での独自のポジション
2. 収益多様化戦略の具体性
3. 海外拠点の強さと戦略
4. デジタル化・DXへの本気度

## 証券

### 証券固有の追加調査観点

**事業構造**
- 引受（IB）業務: IPO・公募増資・債券発行の実績・ランキング
- リサーチ部門: アナリスト数・カバーセクター・レポートの質
- リテール vs 機関投資家: 顧客構成と収益への貢献度
- トレーディング: 自己売買・マーケットメイクの規模

**クエリテンプレート**
```
[企業名] IPO 主幹事 ランキング 2024
[企業名] リサーチ アナリスト 証券アナリストランキング
[企業名] 機関投資家 法人向けサービス 強み
野村証券 vs 大和証券 vs SMBC日興 違い 強み
[企業名] 海外 グローバル 拠点 投資銀行業務
```

**コース別の追加調査**

| コース | 追加で調べること |
|--------|--------------|
| リサーチ | 担当可能セクター・アナリストの外部評価 |
| IB | 主幹事実績・M&Aアドバイザリーの件数 |
| セールス | 機関投資家向けサービスの特徴 |
| トレーディング | 扱う商品の種類・リスク管理体制 |

**差別化ポイントの発見方法**
1. 特定業務（IB / リサーチ）での業界ランキング
2. 外資系証券との差別化（日系の強み）
3. デジタル化・オンライン化への対応

## デジタル・IT（金融機関向け）

### デジタル系コース固有の追加調査観点

**IT戦略・内製化**
- IT子会社の有無と役割: 本体IT部門とIT子会社の分業ライン
- 内製化の実態: 何を内製化しているか（新規開発/保守/AI実装/要件定義）
- ベンダー依存からの脱却度: 中計での内製化方針の明示
- 本体採用IT人材の役割: 企画・上流寄りか開発・実装寄りか

**AI・DX活用の実態**
- AI活用の段階: PoC / 本番稼働 / 定着の度合い
- 適用業務: どの業務ドメインにAIを使っているか
- AIの制約対応: 個人情報・説明可能性・ハルシネーション対策
- DX推進体制: 専門組織の有無・権限・予算規模

**クエリテンプレート**
```
[企業名] IT 内製化 DX推進 中期経営計画 2025
[企業名] IT子会社 情報子会社 本体IT部門 役割 違い
[企業名] AI活用 機械学習 PoC 実装 2024
[企業名] デジタル職 新卒 IT コース 業務内容
[企業名] DX 投資額 エンジニア採用 2024
```

**IT内製化比較マトリクス調査**
- IT戦略子会社の有無・位置づけ
- 内製化への姿勢（中計での明示度）
- AI活用の具体度・実装段階
- 本体採用IT人材の主要ミッション

## データサイエンス職（デジタル系の深掘り）

**データ環境・基盤**
- データ基盤の整備状況: データレイク・DWH・BIツールの整備度
- データの種類と量: 顧客データ・取引データ・市場データの活用状況
- データガバナンス: 個人情報管理・データ品質管理の仕組み

**チーム体制と業務**
- データサイエンスチームの規模: 何人でどんな業務をしているか
- ビジネス側との協働: データサイエンティストの上流関与度
- 成果の定義: KPIの改善・新サービス創出など何を目標にしているか

```
[企業名] データサイエンス 分析 AI チーム 組織
[企業名] データ活用 事例 成果 2024
[企業名] 機械学習 AI 信用リスク 不正検知 活用
```

## DXコンサル・変革推進職（デジタル系の深掘り）

**DX推進体制・実績**
- DX推進専門組織の有無: CDO・DX推進部門の権限・規模
- 変革プロジェクトの具体例: 何を変えたか・何が変わったか
- スピード感: PoC→本番定着までどのくらいかかっているか

**外部コンサル vs 内製の方針**
- 外部コンサルへの依存度
- 自社DX人材の育成方針: 内部育成か外部採用か
- ベンダーとの関係: SIer依存からの脱却状況

```
[企業名] デジタル変革 プロジェクト 成果 事例

```

## ファンドマネージャー・運用職（AM系の深掘り）

**運用プロセスの理解**
- 運用委員会の構造: 意思決定がどう行われるか
- 若手の関与度: 1年目・3年目で実際に何ができるか
- リサーチとファンドマネージャーの連携: どう情報が流れるか
- 外部リサーチの活用度: セルサイドへの依存度vs内製研究

**キャリアパスの具体性**
- ジュニアアナリスト→シニア→PMの道筋: 何年でどう変わるか
- ジョブローテーションの有無
- 外部研修・資格取得支援: CFA・証券アナリスト等の支援

```
[企業名] ファンドマネージャー キャリアパス 若手 1年目
[企業名] 運用プロセス 意思決定 委員会
[企業名] 証券アナリスト CFA 資格支援 取得
```

**差別化ポイントの発見方法**
1. 若手の裁量・意思決定への関与度
2. 独自の運用プロセス・投資哲学
3. キャリア形成の具体的なサポート
