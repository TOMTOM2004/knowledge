# リサーチブリーフ 9軸テンプレート

Step 4 でブリーフを生成する際の軸定義。

---

### 軸1: 事業・ビジネスモデル

- 収益源・主要サービス・事業ポートフォリオ
- 注力している成長領域

### 軸1-B: 収益構造の中身

- 主な収益モデル（例：信託報酬・手数料・スプレッド等）と各収益源の比重
- チャネル別・顧客層別の収益構造（リテール・法人・機関投資家・グループ内）
- 残高連動型か取引連動型かなど、収益構造の特性

### 軸1-C: 顧客基盤の内訳と戦略

- 主要顧客層（個人・企業年金・機関投資家・グループ会社等）
- 各顧客層へのアプローチ方法・強みのチャネル
- 顧客基盤の課題（例：デジタル化・新規顧客開拓）

### 軸1-D: 最重要経営課題（優先順位付き）

- 中期経営計画や経営者発言から読み取れる重点施策（1〜4位の優先順位で整理）
- 数値目標（AUM・売上・利益率等）と現状とのギャップ
- 自分が入社後に関われる可能性のある課題領域

### 軸2: 配属部署・コースの具体業務

- 日常業務・担当領域・アウトプット（レポート・提案書など）
- チーム構造・上流下流での役割
- 他コースとの分業ライン
- **アウトプット→業務KPIの連結**：アウトプット名だけでなく「そのアウトプットが誰のどのKPIを動かすか」まで記載する（例：「要件定義書」→「保険金支払リードタイム短縮」）

**【IT系コースの場合、必ず追加調査する項目】**
- グループIT会社（情報子会社）と本体IT部門の役割分担
- 「内製化」とは具体的に何を内製化しているのか
- 保守運用 vs 変革案件の比率感
- ユーザー部門（業務部門）との接点の強さ
- アジャイル開発の全社導入状況 vs 部分導入状況
- AI活用の実態：PoC段階か / 本番定着か / どの業務領域に適用しているか
- AI活用の制約：個人情報・説明可能性・ハルシネーション対策

### 軸3: 求める人材像

- 採用HP・説明会で語られる人材像のキーワード
- 評価されやすいスキル・思考スタイル・価値観
- このコース固有の期待値（全社的なものと区別して記載）

### 軸4: 志望コース vs 他コースの差別化軸

- このコースを選ぶ積極的な理由（他コースにはない特徴）
- 「なぜこのコースか」設問で使える観点

### 軸5: キャリアパス

- 入社後3〜5年のモデルルート
- ローテーション・専門深化のパターン
- 中長期でのキャリアゴール例

### 軸6: 企業の強み・弱み（絶対評価）

**強み（ESで言及できる知識として）**
- 独自の経営資源・技術・ネットワーク
- 業界内でのポジション・実績

**弱み・課題（貢献可能性の観点として）**
- 業界・企業固有の課題
- 中期経営計画や経営者発言から読み取れる改善課題
- 自分が入社後に貢献できる余地（軸8と連動）

### 軸7: 同業他社比較・「なぜこの企業か」の論拠

**軸7-A: 企業固有言語・思想の抽出（競合比較の前に必ず実施）**

目的: 採用ページ・社員インタビュー・経営者発言のみに現れる「競合が使わない独自フレーズ」を3〜5個発見し、その中から最も強力な1つを深掘りする。

**Step 7-A-1: 固有性候補の収集（WebSearch）**

| 情報源 | 収集クエリ例 |
|--------|------------|
| 採用ページ・社員インタビュー | `<企業名> 採用 社員インタビュー 仕事の意義 大切にしていること` |
| 経営者・役員発言 | `<企業名> CEO 社長 経営哲学 インタビュー 大切にしていること` |
| 企業理念・バリュー | `<企業名> 企業理念 バリュー 行動指針 フィロソフィー` |

収集時の判断基準:
- 競合他社の採用ページにも同じフレーズが出るなら候補から除外する
- 体験記・口コミ（OpenWork等）で「社員に実際に根付いているか」をクロスチェックする
- 経営判断や歴史的経緯と紐づいているフレーズを優先する

**Step 7-A-2: 固有性候補リスト（3〜5個）**

| # | 固有フレーズ・概念 | 競合が使わない理由 | 出典 |
|---|-----------------|-----------------|------|
| 1 | ...             | ...             | ...  |

候補が3個未満の場合は【固有性不足】タグをつけ、OBOGインタビュー等の追加調査を提案する。

**Step 7-A-3: 核心固有性の選定（1つ）と深掘り**

選定基準（優先順）:
1. 自分のエピソード・価値観と最も自然に接続できる
2. 経営判断・歴史的背景から来ており「なぜそうなったか」を説明できる
3. 競合との差別化に直結する

深掘りレポート（必須）:
```
■ 選定した核心固有性: <フレーズ>

【源泉】どの経営判断・歴史的経緯からこの概念が生まれたか
【競合差異】競合が使う表現との違い（一言）
【面接での使い方】「なぜこの企業か」の回答のどの位置に組み込むか（起点/根拠/締め）
【接続エピソード】research_brief.md 軸8の◎・○のどれと接続するか
【注意点】暗記感が出ないための回避方法
```

→ 軸7-3の「②同業他社ではなく、この企業を選ぶ決め手」は7-A-3の核心固有性を起点として組み立てる。

### 軸8: エピソード適合マッピング

社内ナレッジ（ES/素材/）のエピソードリストと、軸3・軸6・軸7を照合する。

**マッチ評価基準**

| 評価 | 定義 |
|------|------|
| ◎ 強くマッチ | 人材像と高い一致。ESの主軸に使える |
| ○ マッチ | おおむね一致。設問によって使える |
| △ 部分的 | 一部は合致するが不足。補強が必要な点を明記 |
| ✕ ミスマッチ | 人材像と合わない。使用を推奨しない理由を記載 |

**留意事項**
- ◎がゼロでも正直に記載する
- △・✕には必ず「何が足りないか」を一行で添える

**主武器順位（必ず記載する）：**
```
■ <コース名>向け 主武器順位（面接で使う順）：
1. 主軸（最初に出す）: <エピソード名> → 理由一行
2. 補強: <エピソード名> → 理由一行
3. 差別化: <エピソード名> → 理由一行
4. テーマ深掘り / 仕事哲学: <エピソード名> → 理由一行
```

### 軸9: 面接対策・詰められ論点・逆質問候補

**9-1. 詰められ論点の想定**
- 「なぜこの業界か」「なぜこの企業か」「なぜこのコースか」の3段の深掘り想定Q&A
- 志望理由の弱点を先に洗い出し、防衛ラインを設定
- エピソードへの深掘り想定

**【IT系コースの場合、9-1に必ず追加する6段反論軸】**
「なぜIT全般か→なぜ事業会社ITか→なぜこの業界のITか→なぜこの業種か→なぜこの企業か→なぜこのコースか」の6段

**9-2. 逆質問候補**
- 業務理解・成長環境・戦略理解・社風の4カテゴリから最低2問ずつ
- 「調べればわかる」質問は使わない

**9-3. 自分のエピソードと企業接続マップ**
- 各◎・○エピソードと「企業の何」が接続するかを一行で記載
