# 調査品質チェック・追加調査プロセス

Step 7: 保存後に必ず実行する品質チェック手順。

---

## 7-1. 機械チェック

```bash
python /Users/ishidatomonori/Desktop/knowledge/tools/checks/research_checker.py "<企業名>"
```

結果を確認し、ERRORがあれば即座にユーザーへ提示する。

## 7-2. 調査ギャップ分析

`research-gap-reviewer` エージェントの手順に従い、以下を実施する:

1. research_brief.md の9軸カバレッジを評価する（◎/○/△/✗）
2. 不足項目を優先度で分類する:
   - **P0（最優先）**: 「なぜこの会社か」「競合との差別化」の根拠が不足 → ES・面接の核心に直結
   - **P1（重要）**: 中計・キャリアパス・求める人材像の具体性不足 → 面接で突っ込まれる可能性高
   - **P2（補足）**: OBOGインタビュー・組織詳細など → あると良いが必須ではない

## 7-3. P0・P1不足への追加調査（自動実行）

P0またはP1に分類された不足項目がある場合、確認なしで以下を実行する:

1. 不足項目ごとに具体的な調査クエリを設計する:
   ```
   P0「競合差別化の根拠不足」→「<企業名> vs <競合名> 強み 差別化 2025」
   P1「中計の注力領域不足」→「<企業名> 中期経営計画 2025 重点施策」
   P1「キャリアパス不足」→「<企業名> <コース名> キャリアパス 昇進 ローテーション」
   ```
2. WebSearch / WebFetch で追加調査を実行する（P0から順に最大5クエリ）
3. 取得した情報をresearch_brief.mdの該当軸に追記・更新する
4. 【要補完】タグがあった箇所のうち解消できたものを削除する

P2のみの不足であれば追加調査は行わず、不足項目リストの提示のみ行う。

## 品質ガードレール

- 未確認の数値・シェア・順位は使わない。使う場合は `【要確認】` タグをつける
- Web情報はURLとともに記録し、情報の出所を明示する
- 弱み・課題は批判的トーンではなく「貢献余地」として中立に記述する
- エピソードのマッチ評価は過度に楽観的にしない
